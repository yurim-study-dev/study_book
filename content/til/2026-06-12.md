## 오전/오후: DDL 심화 - 2.1

### ALTER TABLE - 테이블 구조 변경

이미 만들어진 테이블의 구조를 변경할 수 있다.

```sql
-- 컬럼 추가
ALTER TABLE member ADD phone VARCHAR(20);

-- 컬럼 수정 (타입 변경 + 제약 조건 추가)
ALTER TABLE member MODIFY phone VARCHAR(30) NOT NULL;

-- 컬럼 삭제
ALTER TABLE member DROP COLUMN phone;
```

---

### TRUNCATE vs DROP 비교

| 구분 | 설명 | 롤백 |
|------|------|------|
| `DROP TABLE 테이블명` | 테이블 구조 + 데이터 모두 삭제 | 불가 |
| `TRUNCATE TABLE 테이블명` | 데이터만 전체 삭제, 구조(스키마)는 유지 | 불가 |
| `DELETE FROM 테이블명` | 데이터만 삭제 (WHERE로 조건 가능) | 가능 |

```sql
TRUNCATE TABLE post;    -- 데이터 초기화, 구조 유지. DELETE보다 빠름
```

### FK 외래 키 참조 옵션 (ON DELETE)

부모 테이블 레코드가 삭제될 때 자식 테이블을 어떻게 처리할지 지정한다.

| 옵션 | 설명 |
|------|------|
| `ON DELETE CASCADE` | 부모 삭제 시 자식 레코드도 함께 삭제 |
| `ON DELETE SET NULL` | 부모 삭제 시 자식의 FK 값을 NULL로 변경 |
| `ON DELETE RESTRICT` | 자식이 참조 중인 부모는 삭제 불가 (기본값) |

```sql
-- post 테이블: 회원 삭제 시 게시글의 member_id를 NULL로 변경
CREATE TABLE post (
    id INT AUTO_INCREMENT PRIMARY KEY,
    member_id INT,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (member_id) REFERENCES member(id) ON DELETE SET NULL
);

-- reply 테이블: 게시글 삭제 시 댓글도 함께 삭제
CREATE TABLE reply (
    id INT AUTO_INCREMENT PRIMARY KEY,
    post_id INT NOT NULL,
    member_id INT NOT NULL,
    content TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (post_id) REFERENCES post(id) ON DELETE CASCADE,
    FOREIGN KEY (member_id) REFERENCES member(id) ON DELETE CASCADE
);
```

## 오전/오후: DML 심화 - 2.2

### INSERT - 삽입 방법 3가지

```sql
-- 1) 전체 컬럼 값 삽입 (컬럼명 생략)
-- AUTO_INCREMENT는 NULL, DEFAULT 컬럼은 DEFAULT 키워드 사용
INSERT INTO member
    VALUES (NULL, 'haru@gmail.com', 'pwd123', '하루', DEFAULT);

-- 2) 특정 컬럼만 지정하여 삽입
INSERT INTO member (email, password, name)
    VALUES ('namu@gmail.com', 'pwd789', '나무');

-- 3) 다중 행 삽입 (한 번에 여러 개)
INSERT INTO member (email, password, name, phone) VALUES
    ('harong@gmail.com', 'pwd012', '하롱이', '01022223333'),
    ('yong@gmail.com', 'pwd456', '용쌤', '01033334444');
```

> 다중 행 삽입은 네트워크 트래픽을 줄이고 대용량 데이터를 빠르게 넣을 때 유용하다.

### SELECT - 조회 옵션

```sql
-- 특정 컬럼 조회
SELECT name, created_at FROM member;

-- 전체 컬럼 조회
SELECT * FROM member;

-- 별칭(Alias) 지정 - AS 생략 가능
SELECT name, created_at AS 가입일 FROM member;

-- 중복 제거 조회
SELECT DISTINCT member_id FROM post;
```

---

### UPDATE / DELETE 주의사항

```sql
-- WHERE 없으면 전체 행이 수정/삭제됨 → 반드시 SELECT로 먼저 확인!
UPDATE member SET name = '김하루' WHERE id = 1;
DELETE FROM reply WHERE id = 1;

-- ON DELETE CASCADE 적용 예시
-- 2번 게시글 삭제 시 연관 댓글도 자동 삭제됨
DELETE FROM post WHERE id = 2;
```

## 오전/오후: SELECT 심화 - 2.3

### WHERE - 조건 검색

```sql
-- 비교 연산자 (=, <>, !=, >, <, >=, <=)
SELECT id, name FROM member WHERE email = 'haru@gmail.com';

-- LIKE - 패턴 매칭
--   % : 길이 제한 없는 모든 문자열
--   _ : 문자 1개
SELECT id, title FROM post WHERE title LIKE '%게시글%';
SELECT id, name FROM member WHERE name LIKE '하_';   -- '하'로 시작하는 두 글자

-- IN - 여러 값 중 하나와 일치
SELECT id, title, member_id FROM post WHERE member_id IN (1, 3, 5);

-- BETWEEN A AND B - 범위 검색
SELECT id, name, created_at FROM member
    WHERE created_at BETWEEN '2026-06-01' AND '2026-06-10 23:59:59';

-- IS NULL / IS NOT NULL
SELECT id, name FROM member WHERE phone IS NULL;
```

---

### ORDER BY - 결과 정렬

```sql
-- 내림차순 (최신 순)
SELECT id, email, name, created_at FROM member
    ORDER BY created_at DESC;

-- 다중 컬럼 정렬: 이름 오름차순 → 동일 이름은 가입일 내림차순
SELECT id, name, created_at FROM member
    ORDER BY name ASC, created_at DESC;
```

| 정렬 방식 | 키워드 | 설명 |
|-----------|--------|------|
| 오름차순 | `ASC` (기본값) | 작은 값 → 큰 값, 과거 → 최신 |
| 내림차순 | `DESC` | 큰 값 → 작은 값, 최신 → 과거 |

---

### LIMIT - 행 개수 제한 / 페이징

```sql
-- 최근 게시글 5개만 조회
SELECT id, title, created_at FROM post
    ORDER BY created_at DESC
    LIMIT 5;

-- 페이징: 1페이지(1~10번째)
SELECT id, title, created_at FROM post ORDER BY created_at DESC LIMIT 0, 10;

-- 페이징: 2페이지(11~20번째)
SELECT id, title, created_at FROM post ORDER BY created_at DESC LIMIT 10, 10;
```

> `LIMIT 시작위치, 개수` → 시작위치는 0부터 시작한다.

## 오후: DCL (Data Control Language) - 2.3

DCL은 데이터베이스 사용자 계정 생성 및 권한을 관리하는 SQL이다.

```sql
-- 사용자 생성
CREATE USER '유저명'@'호스트' IDENTIFIED BY '비밀번호';

-- 권한 부여 (GRANT)
GRANT ALL PRIVILEGES ON 데이터베이스명.* TO '유저명'@'호스트';

-- 권한 회수 (REVOKE)
REVOKE ALL PRIVILEGES ON 데이터베이스명.* FROM '유저명'@'호스트';

-- 권한 적용 새로고침
FLUSH PRIVILEGES;
```

## 배운점

1. ✅ `ALTER TABLE`로 테이블 구조를 언제든지 변경할 수 있다 (컬럼 추가/수정/삭제).
2. ✅ FK의 `ON DELETE CASCADE`는 부모 삭제 시 자식도 연쇄 삭제, `SET NULL`은 NULL 처리된다.
3. ✅ 다중 행 INSERT는 한 번의 쿼리로 대량 데이터를 빠르게 삽입할 수 있다.
4. ✅ `LIKE`의 `%`는 모든 문자열, `_`는 정확히 1개 문자를 의미한다.
5. ✅ `IN`, `BETWEEN`, `IS NULL`로 다양한 조건 필터링이 가능하다.
6. ✅ `ORDER BY`는 여러 컬럼을 지정해 1차/2차 정렬을 설정할 수 있다.
7. ✅ `LIMIT 시작위치, 개수`로 페이징 처리를 구현할 수 있다 (0부터 시작).
8. ✅ UPDATE/DELETE 전에 SELECT로 대상 레코드를 먼저 확인하는 습관이 중요하다.

## 오늘 핵심 정리

```
DDL 심화
  ALTER TABLE 테이블명 ADD    컬럼명 타입    → 컬럼 추가
  ALTER TABLE 테이블명 MODIFY 컬럼명 타입    → 컬럼 수정
  ALTER TABLE 테이블명 DROP COLUMN 컬럼명   → 컬럼 삭제
  TRUNCATE TABLE 테이블명                   → 데이터만 초기화 (구조 유지)

FK ON DELETE 옵션
  CASCADE   → 부모 삭제 시 자식도 삭제
  SET NULL  → 부모 삭제 시 자식 FK를 NULL로
  RESTRICT  → 자식이 참조 중이면 부모 삭제 불가 (기본값)

SELECT 핵심 절
  WHERE   → 조건 필터링 (LIKE, IN, BETWEEN, IS NULL)
  ORDER BY → 정렬 (ASC 오름차순 / DESC 내림차순)
  LIMIT   → 행 개수 제한 / 페이징 처리

DCL
  GRANT  → 권한 부여
  REVOKE → 권한 회수
```
