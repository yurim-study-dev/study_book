## 1. 기본 클래스 구성: Car 클래스 (ex01)

### 목표
필드와 메서드를 정의하고 객체를 생성하여 사용하는 실습

### Car 클래스 구조

```java
public class Car {
    // 필드 (객체의 속성)
    String model;      // 자동차 모델명
    int speed;         // 현재 속도
    int gear;          // 현재 기어
    
    // 메서드 (객체의 행위)
    void speedUp()      { ... }    // 가속
    void speedDown()    { ... }    // 감속
    void gearUp()       { ... }    // 기어 상향
    void gearDown()     { ... }    // 기어 하향
    void start()        { ... }    // 출발
    void stop()         { ... }    // 정지
}
```

### 필드 자동 초기화

```java
Car mycar = new Car();
System.out.println(mycar.speed);  // 0 (자동으로 초기화됨)
System.out.println(mycar.gear);   // 0
System.out.println(mycar.model);  // null
```

**중요:** 클래스 필드는 선언만 해도 자동으로 기본값으로 초기화됩니다.
- 숫자형: 0
- 참조형: null

### 메서드 구현 분석

#### 1️⃣ 속도 관련 메서드
```java
void speedUp() {
    speed += 10;
    System.out.println("가속합니다. 현재 속도는 " + speed + "km/h 입니다.");
}

void speedDown() {
    speed -= 10;
    System.out.println("감속합니다. 현재 속도는 " + speed + "km/h 입니다.");
}
```

#### 2️⃣ 기어 관련 메서드
```java
void gearUp() {
    gear++;
    System.out.println("기어를 1단 올려서 " + gear + "단이 되었습니다.");
}
```

#### 3️⃣ 복합 메서드: start() - 메서드 체이닝
```java
void start() {
    System.out.println("출발합니다.");
    gearUp();      // 다른 메서드 호출
    speedUp();     // 다른 메서드 호출
}
```

#### 4️⃣ 루프를 활용한 메서드: stop()
```java
void stop() {
    System.out.println("멈춥니다.");
    gear = 0;
    
    // 속도가 0이 될 때까지 반복 감속
    while (speed > 0) {
        speedDown();
    }
}
```

### 사용 예시 (CarTest.java)

```java
Car mycar = new Car();
mycar.model = "부가티";    // 필드에 값 할당

mycar.start();             // 출발 (기어 올리고 가속)
mycar.speedUp();           // 여러 번 가속
mycar.speedUp();
mycar.speedUp();

mycar.gearUp();            // 기어 올리기
mycar.speedUp();
mycar.speedUp();

mycar.stop();              // 정지 (속도가 0이 될 때까지 반복 감속)
```

**출력:**
```
출발합니다.
기어를 1단 올려서 1단이 되었습니다.
가속합니다. 현재 속도는 10km/h 입니다.
...
멈춥니다.
감속합니다. 현재 속도는 0km/h 입니다.
```

## 2. 생성자와 생성자 오버로딩: Car 클래스 (ex02)

### 목표
생성자를 정의하고 오버로딩하여 객체 초기화 방법을 다양화

### 생성자란?

**생성자(Constructor)**
- new 연산자로 객체가 생성될 때 **가장 먼저 자동 호출**되는 메서드
- 객체의 초기화를 담당
- 반환 타입이 없음 (void도 쓰지 않음)
- 메서드명 = 클래스명 (대소문자 정확히 일치)

### 기본 생성자

```java
public class Car {
    // ...필드...
    
    // 기본 생성자 (매개변수 없음)
    Car() {
        System.out.println("Car() 생성자 호출됨.");
    }
}

// 사용
Car car1 = new Car();  // "Car() 생성자 호출됨." 출력
```

### 생성자 오버로딩

같은 이름이지만 **매개변수가 다른 여러 생성자를 정의**할 수 있습니다.

```java
public class Car {
    String model;
    int speed;
    int gear;
    
    // 생성자 1: 매개변수 없음
    Car() {
        System.out.println("Car() 생성자 호출됨.");
    }
    
    // 생성자 2: String 매개변수
    Car(String model) {
        System.out.println("Car(String model) 생성자 호출됨.");
        this.model = model;
    }
    
    // 생성자 3: String, int 매개변수
    Car(String model, int speed) {
        System.out.println("Car(String model, int speed) 생성자 호출됨.");
        this.model = model;
        this.speed = speed;
    }
}
```

### 생성자 오버로딩 사용 예

```java
Car car1 = new Car();                    // 생성자 1 호출
Car car2 = new Car("소나타");            // 생성자 2 호출
Car car3 = new Car("아반떼", 100);       // 생성자 3 호출
```

### this 키워드

**this**: 자기 자신(현재 객체)을 가리키는 참조

```java
Car(String model) {
    this.model = model;  // 매개변수 model과 필드 model을 구분
}
```

**this의 역할:**
- 매개변수와 필드 이름이 같을 때 **구분**
- 가독성 향상
- 자신의 객체에 명시적으로 접근

## 3. 간단한 클래스: Person, Member, Book

### Person 클래스 - 기본 필드와 메서드

```java
public class Person {
    String name;
    int age;
    
    void introduce() {
        System.out.println("내 이름은 " + name + "이고, 나이는 " + age + "살입니다.");
    }
}
```

**사용 예시:**
```java
Person p = new Person();
p.name = "하루";
p.age = 6;
p.introduce();  // 내 이름은 하루이고, 나이는 6살입니다.
```

### MemberTest 클래스 - 생성자 활용

```java
public class MemberTest {
    String id;
    
    // 생성자: 아이디로 멤버 생성
    MemberTest(String id) {
        this.id = id;
    }
    
    public static void main(String[] args) {
        MemberTest member = new MemberTest("yong");
        System.out.println("회원 아이디: " + member.id);
    }
}
```

### BookTest 클래스 - 생성자 오버로딩과 this()

```java
class Book {
    String title;
    String author;
    
    // 기본 생성자: this()로 다른 생성자 호출
    Book() {
        this("제목미상", "작가미상");
    }
    
    // 매개변수 있는 생성자
    Book(String title, String author) {
        this.title = title;
        this.author = author;
    }
}

public class BookTest {
    public static void main() {
        Book book1 = new Book();
        Book book2 = new Book("자바를 자바라!", "용쌤");
        
        System.out.println("제목: " + book1.title + ", 작가: " + book1.author);
        // 제목: 제목미상, 작가: 작가미상
        
        System.out.println("제목: " + book2.title + ", 작가: " + book2.author);
        // 제목: 자바를 자바라!, 작가: 용쌤
    }
}
```

**this() 용법:**
```java
Book() {
    this("제목미상", "작가미상");  // Book(String, String) 생성자 호출
}
```
- 다른 생성자를 호출할 때 사용
- **반드시 생성자의 첫 줄에만 위치 가능**
- 생성자 오버로딩 시 코드 중복을 줄일 수 있음

## 4. 패키지와 임포트: 클래스의 조직화

### 패키지란?

**패키지(Package)**: 관련된 클래스들을 조직화하고 그룹화하는 방법

```
패키지 구조:
ch11/
├── mycar/
│   ├── ex01/
│   │   ├── Car.java
│   │   └── CarTest.java
│   └── ex02/
│       ├── Car.java
│       └── CarTest.java
├── pkg/
│   ├── a/
│   │   └── SomeClass.java
│   └── b/
│       └── ExternalClassTest.java
└── lms/
    ├── Test.java
    ├── 강의실.java
    ├── 과목.java
    ├── 학생.java
    └── 교수.java
```

### 패키지 선언

```java
// SomeClass.java
package ch11.pkg.a;

public class SomeClass {
    public SomeClass() {
        System.out.println("SomeClass() 생성자 호출");
    }
}
```

**규칙:**
- 패키지 선언은 소스 코드의 **첫 줄**
- 폴더 구조와 패키지명이 일치해야 함

### 임포트 (Import)

다른 패키지의 클래스를 사용하려면 import 필요

```java
// ExternalClassTest.java
package ch11.pkg.b;

import ch11.mycar.ex02.Car;      // 다른 패키지의 Car 클래스 임포트
import ch11.pkg.a.SomeClass;     // 다른 패키지의 SomeClass 임포트

public class ExternalClassTest {
    public static void main() {
        SomeClass s = new SomeClass();
        // Car c = new Car();  // 주석 처리됨
    }
}
```

**임포트 문법:**
```java
import 패키지명.클래스명;
import ch11.pkg.a.SomeClass;
```

**같은 패키지 내 클래스는 임포트 불필요:**
```java
// ch11.mycar.ex02.CarTest.java에서
Car car = new Car();  // import 필수 없음 (같은 패키지)
```

## 5. 핵심 개념 정리

### 필드의 자동 초기화

```
데이터 타입          기본값
─────────────────────────────
boolean             false
byte, short, int    0
long                0L
float               0.0f
double              0.0
char                ' ' (공백 문자)
참조형(String 등)   null
```

### 생성자와 초기화 비교

| 구분 | 필드만 선언 | 생성자 | 이점 |
|------|----------|--------|------|
| 초기값 설정 | 자동(기본값) | 명시적 | 의도한 값으로 초기화 |
| 객체 생성 | 가능 | 필요 시 | 유효한 상태 보장 |
| 코드 안정성 | 낮음 | 높음 | 버그 감소 |

### 메서드에서 다른 메서드 호출

```java
void start() {
    System.out.println("출발합니다.");
    gearUp();      // 같은 클래스의 다른 메서드 호출
    speedUp();
}
```

**특징:**
- 코드 재사용
- 로직 분리
- 유지보수 용이

### 루프를 활용한 필드 처리

```java
void stop() {
    gear = 0;
    while (speed > 0) {
        speedDown();  // 속도가 0이 될 때까지 반복 감속
    }
}
```

## 배운점

1. ✅ 클래스는 필드와 메서드로 구성되고, 필드는 자동으로 초기화된다.
2. ✅ 객체는 new 연산자로 생성되고, 도트(.)로 필드와 메서드에 접근한다.
3. ✅ 생성자는 객체 생성 시 자동으로 호출되어 초기 설정을 담당한다.
4. ✅ 생성자 오버로딩으로 다양한 방식의 객체 초기화가 가능하다.
5. ✅ this는 자신의 객체를 가리키며, 매개변수와 필드를 구분할 때 유용하다.
6. ✅ this()는 다른 생성자를 호출하고, 반드시 첫 줄에 위치해야 한다.
7. ✅ 패키지는 클래스들을 조직화하고, import로 다른 패키지의 클래스를 사용한다.
8. ✅ 객체 조합(Object Composition)으로 클래스 간의 관계를 표현할 수 있다.
9. ✅ 메서드 내에서 다른 메서드를 호출하면 코드 재사용성이 높아진다.
10. ✅ 복잡한 시스템도 여러 클래스로 분산하여 설계할 수 있다.
