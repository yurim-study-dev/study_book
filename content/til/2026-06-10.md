## 오전: String 클래스 - 14.3

### 14.3.1 String의 3가지 핵심 특징

#### 1) 불변성 (Immutability)

String 객체는 한 번 생성되면 내부 값을 절대 바꿀 수 없다.

```java
// 내부적으로: private final byte[] value (Java 9 이후)
// final이므로 수정 불가
String str = "Hello Java";
str.replace("Java", "World"); // 새 객체 반환, 원본 str은 그대로
System.out.println(str); // Hello Java (원본 불변)
```

#### 2) 문자열 상수 풀 (String Constant Pool)

힙 메모리 내부의 특수한 공간. JVM이 메모리 절약을 위해 관리.

```java
String s1 = "Hello";         // 상수 풀에서 찾고 없으면 등록 → 주소 0x12
String s2 = "Hello";         // 이미 있음 → 0x12 재사용
String s3 = new String("Hello"); // 힙에 새 객체 생성 → 주소 0x34

System.out.println(s1 == s2);      // true  (같은 상수 풀 주소)
System.out.println(s1 == s3);      // false (주소가 다름)
System.out.println(s1.equals(s3)); // true  (내용이 같음)
```

> 문자열 내용 비교는 반드시 `equals()` 사용! `==`는 주소 비교임

#### 3) 문자열 결합 연산의 부작용

루프에서 `+` 연산으로 문자열을 계속 합치면 심각한 성능 저하 발생.

```java
// ❌ 나쁜 방식: 매 반복마다 새 객체 생성 → 가비지 폭증
String str = "";
for (int i = 0; i < 100; i++) {
    str = str + "Hello "; // 루프마다 새 String 객체가 힙에 쌓임
}

// ✅ 좋은 방식: 버퍼 하나를 계속 재사용
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100; i++) {
    sb.append("Hello ");  // 버퍼 내부를 수정 (새 객체 생성 없음)
}
```

### 14.3.2 String 주요 메서드

| 메서드 | 설명 | 예시 결과 |
|--------|------|---------|
| `length()` | 문자열 길이 반환 | `"Hello Java".length()` → `10` |
| `charAt(int index)` | 특정 인덱스의 문자 반환 | `"Hello Java".charAt(6)` → `'J'` |
| `substring(begin, end)` | begin ~ end 직전까지 추출 | `"Hello Java".substring(6, 10)` → `"Java"` |
| `substring(begin)` | begin부터 끝까지 추출 | `"Hello Java".substring(4)` → `"o Java"` |
| `equals(obj)` | 내용 동등 비교 | `"Hello".equals("Hello")` → `true` |
| `indexOf(str)` | 문자열이 처음 나오는 인덱스 | `"Hello Java".indexOf("Java")` → `6` |
| `indexOf(str)` | 없으면 -1 반환 | `"Hello Java".indexOf("java")` → `-1` |
| `replace(target, replacement)` | 문자열 치환 후 새 객체 반환 | `"Hello Java".replace("Java", "자바")` → `"Hello 자바"` |
| `split(regex)` | 구분자 기준 분할 → 배열 반환 | `"a b c".split(" ")` → `["a","b","c"]` |
| `toLowerCase()` | 소문자 변환 | `"Hello".toLowerCase()` → `"hello"` |
| `toUpperCase()` | 대문자 변환 | `"Hello".toUpperCase()` → `"HELLO"` |
| `trim()` | 앞뒤 공백 제거 | `"  Hello  ".trim()` → `"Hello"` |
| `startsWith(prefix)` | 특정 문자열로 시작 여부 | `"Hello Java".startsWith("Hello")` → `true` |

---

### 14.3.3 실습 예제: StringMethodTest.java

```java
String str = "Hello Java";

// 대소문자 구분 없이 indexOf 활용
// toLowerCase()로 모두 소문자 통일 후 검색
System.out.println(str.toLowerCase().indexOf("java".toLowerCase())); // 6
System.out.println(str.toLowerCase().indexOf("Java".toLowerCase())); // 6
System.out.println(str.toLowerCase().indexOf("jaVA".toLowerCase())); // 6

// trim() 앞뒤 공백 제거
String str3 = "   Hello Java World  ";
System.out.println("[" + str3 + "]");        // [   Hello Java World  ]
System.out.println("[" + str3.trim() + "]"); // [Hello Java World]

// startsWith() vs indexOf() == 0 비교
System.out.println(str.indexOf("Hello") == 0); // true (방법 1)
System.out.println(str.startsWith("Hello"));   // true (방법 2, 더 직관적)
```

## 오전: StringBuffer와 StringBuilder 클래스 - 14.4

### 14.4.1 StringBuffer vs StringBuilder

| 구분 | StringBuffer | StringBuilder |
|------|-------------|--------------|
| **동기화** | ✅ 있음 (멀티스레드 안전) | ❌ 없음 (싱글스레드 전용) |
| **속도** | 상대적으로 느림 | 상대적으로 빠름 |
| **권장 환경** | 멀티스레드 | 싱글스레드 (대부분의 경우) |

두 클래스의 **API는 동일** → 상황에 맞게 선택만 하면 됨.

**핵심 차이:** String과 달리 내부 버퍼(byte[])를 직접 수정 → 새 객체 생성 없음!

---

### 14.4.2 주요 생성자 (StringBuffer 기준)

```java
StringBuffer sb1 = new StringBuffer();         // 기본 크기 16
StringBuffer sb2 = new StringBuffer(1000);     // 초기 크기 지정 (성능 최적화)
StringBuffer sb3 = new StringBuffer("Java");  // 문자열로 초기화 (크기: "Java".length() + 16)
```

---

### 14.4.3 주요 메서드

| 메서드 | 설명 |
|--------|------|
| `append(str)` | 끝에 문자열 추가, 자신을 반환 |
| `insert(offset, str)` | offset 위치에 삽입, 자신을 반환 |
| `delete(start, end)` | start ~ end 직전까지 삭제, 자신을 반환 |
| `reverse()` | 문자열 순서 반전, 자신을 반환 |
| `toString()` | 일반 String 객체로 변환 |

```java
    sb2.append("Hello ");
```

## 오후: 래퍼 클래스 - 14.6

### 래퍼 클래스란?

기본 자료형(Primitive Type)을 객체로 다루어야 할 때 사용하는 클래스.
`Object[]`에 기본 타입을 넣거나, `parseInt()` 같은 유틸리티 메서드를 쓸 때 필요.

| 기본 타입 | 래퍼 클래스 |
|---------|-----------|
| `byte` | `Byte` |
| `short` | `Short` |
| `int` | `Integer` |
| `long` | `Long` |
| `float` | `Float` |
| `double` | `Double` |
| `char` | `Character` |
| `boolean` | `Boolean` |

---

### 14.6.1 박싱과 언박싱

```java
// 명시적 박싱/언박싱 (예전 방식)
Integer num1 = Integer.valueOf(100); // int → Integer (박싱)
int value1 = num1.intValue();        // Integer → int (언박싱)

// 오토 박싱/언박싱 (Java 5 이후, 자동 변환)
Integer num2 = 100;   // 오토 박싱
int value2 = num2;    // 오토 언박싱

// Object[]에 기본 타입 저장 가능 (오토 박싱 덕분)
Object[] oArr = {"Hello", "Java", 4}; // 4 → Integer로 자동 변환
```

---

### 14.6.2 주요 메서드

| 메서드 | 설명 |
|--------|------|
| `Integer.parseInt(String s)` | 문자열 → `int` 변환 |
| `Double.parseDouble(String s)` | 문자열 → `double` 변환 |
| `Boolean.parseBoolean(String s)` | 문자열 → `boolean` 변환 |
| `Integer.valueOf(int i)` | `int` → `Integer` 객체 변환 |

---

### 14.6.3 실습 예제: WrapperTest.java

```java
// 1. 문자열 파싱
int num1 = Integer.parseInt("100");
double num2 = Double.parseDouble("3.14");
boolean bool = Boolean.parseBoolean("true");
System.out.println(num1); // 100
System.out.println(num2); // 3.14
System.out.println(bool); // true

// 2. valueOf(): 기본 타입 → 래퍼 객체
Integer wrapperInt = Integer.valueOf(500);
System.out.println(wrapperInt); // 500

// 3. 오토 박싱 / 오토 언박싱
Integer boxedNum = 200;  // 오토 박싱
int unboxedNum = boxedNum; // 오토 언박싱
System.out.println(unboxedNum); // 200
```

## 오후: Scanner 클래스 - 14.8

### Scanner 클래스 특징

- 공백(스페이스, 탭)이나 개행(줄바꿈)을 구분자로 토큰을 읽어옴
- 파일로부터 읽을 경우 반드시 `close()`로 스트림 닫아야 함 (리소스 누수 방지)

### 14.8.1 주요 메서드

| 메서드 | 설명 |
|--------|------|
| `next()` | 공백 이전까지의 단어를 `String`으로 반환 |
| `nextLine()` | 줄바꿈 전까지 한 줄 전체를 `String`으로 반환 |
| `nextInt()` | 다음 토큰을 `int`로 파싱하여 반환 |
| `nextDouble()` | 다음 토큰을 `double`로 파싱하여 반환 |

---

### 14.8.2 실습 예제: ScannerTest.java

```java
String input = "Hello 123 3.14 java";
Scanner scanner = new Scanner(input);

System.out.println("첫번째 토큰: " + scanner.next());       // Hello
System.out.println("두번째 토큰: " + scanner.nextInt());    // 123
System.out.println("세번째 토큰: " + scanner.nextDouble()); // 3.14

if (scanner.hasNext()) {
    System.out.println("네번째 토큰: " + scanner.next()); // java
}

if (scanner.hasNext()) {
    System.out.println("다섯번째 토큰: " + scanner.next()); // 실행 안됨 (토큰 없음)
}
```

> 순서대로 읽기 때문에 `nextInt()` 자리에서 문자열이 오면 예외 발생 — 타입 순서를 맞춰야 함

## 오후: Random 클래스 - 14.9

### Random vs Math.random() 비교

| 구분 | `Math.random()` | `Random` 클래스 |
|------|----------------|----------------|
| 반환 타입 | `double` (0.0 ~ 1.0 미만) 고정 | `int`, `double`, `boolean` 등 다양 |

| 씨드 지정 | 불가 | `new Random(seed)` 로 가능 |

---

```java
Random random = new Random();       // 매번 다른 난수 (시스템 시간을 씨드로 사용)
Random random = new Random(12345);  // 항상 동일한 순서의 난수 (씨드 고정)
```

---

## 오후: 날짜와 시간 클래스 - 14.10

### java.time 패키지가 생긴 이유

기존 `Date`, `Calendar` 클래스의 문제점:
- 월(Month)이 0부터 시작 → 1월이 `0`, 12월이 `11` (매우 혼란스러움)
- 값 변경이 자유로워 멀티스레드 환경에서 안전하지 않음

→ Java 8부터 `java.time` 패키지로 대체. 현재는 레거시 코드 외에는 거의 안 씀.

---

### 14.10.1 주요 클래스

| 클래스 | 용도 |
|--------|------|
| `LocalDate` | 날짜만 (연·월·일) |
| `LocalTime` | 시간만 (시·분·초) |
| `LocalDateTime` | 날짜 + 시간 모두 |
| `DateTimeFormatter` | 날짜/시간 → 문자열 포맷 변환 |

---

### 14.10.2 DateTimeFormatter 패턴

| 패턴 | 의미 | 예시 |
|------|------|------|
| `yyyy` | 4자리 연도 | `2026` |
| `MM` | 2자리 월 | `06` |
| `dd` | 2자리 일 | `10` |
| `HH` | 24시간제 시 | `14` |
| `mm` | 분 | `30` |
| `ss` | 초 | `05` |
| `E` | 요일 | `수` |

---

### 14.10.3 실습 예제: TimeTest.java

```java
LocalDate today = LocalDate.now();
LocalDateTime now = LocalDateTime.now();

// 날짜 연산
LocalDate futureDate = today.plusDays(7);
LocalDate pastDate = today.minusDays(7);

// 개별 필드 추출
System.out.println(today.getYear());       // 2026
System.out.println(today.getMonthValue()); // 6 (1~12, Calendar처럼 +1 불필요!)
System.out.println(today.getDayOfMonth()); // 10

// 커스텀 포맷
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy년 MM월 dd일");
System.out.println(today.format(formatter)); // 2026년 06월 10일

// 부트캠프 날짜 계산 (toEpochDay 활용)
LocalDate startDate = LocalDate.of(2026, 5, 14);
LocalDate finishDate = LocalDate.of(2026, 11, 16);

System.out.println("시작일로부터 지난 일수: " + (today.toEpochDay() - startDate.toEpochDay()));
System.out.println("수료일까지 남은 일수: " + (finishDate.toEpochDay() - today.toEpochDay()));
System.out.println("총 수업 기간: " + (finishDate.toEpochDay() - startDate.toEpochDay()));
```

> `toEpochDay()`: 1970-01-01 기준으로 경과한 일수를 반환 → 두 날짜의 차이를 쉽게 계산 가능

## 오후: ArrayList 클래스 - 14.11

### ArrayList vs 배열 비교

| 구분 | 배열 (`int[]`) | `ArrayList<E>` |
|------|--------------|----------------|
| 크기 | 고정 | 동적 (자동 확장) |
| 타입 | 기본형 가능 | 객체(참조형)만 가능 |
| 길이 확인 | `.length` | `.size()` |
| 요소 접근 | `arr[i]` | `list.get(i)` |

---

### 14.11.1 제네릭(Generic)

꺾쇠 `<E>` 안에 저장할 타입을 지정 → 잘못된 타입 저장 시 컴파일 에러로 방지

```java
ArrayList<String> list = new ArrayList<>(); // String만 저장 가능
list.add("Apple");
list.add(123); // ❌ 컴파일 에러!
```

```java
ArrayList<String> list = new ArrayList<>(2); // 초기 용량 2

```

## 배운점

1. ✅ 자바 표준 API는 공식 문서(docs.oracle.com)에서 확인할 수 있다.
2. ✅ `java.lang` 패키지는 import 없이 사용 가능, `java.util` / `java.time`은 import 필요.
3. ✅ String은 불변 객체 → `replace()` 등은 원본을 바꾸지 않고 새 객체를 반환한다.
4. ✅ 리터럴 `"Hello"`와 `new String("Hello")`는 주소가 다르므로 `==` 비교는 위험하다.
5. ✅ 문자열 내용 비교는 항상 `equals()`를 사용해야 한다.
6. ✅ 루프 내 `+` 연산은 가비지 객체를 폭증시켜 성능을 저하시킨다.
7. ✅ 빈번한 문자열 편집에는 `StringBuilder` (싱글스레드) 또는 `StringBuffer` (멀티스레드)를 사용한다.
8. ✅ 래퍼 클래스로 기본 타입을 객체로 다룰 수 있고, 오토 박싱/언박싱으로 자동 변환된다.
9. ✅ `Integer.parseInt()` 등 파싱 메서드로 문자열을 기본 타입으로 변환할 수 있다.
10. ✅ `Math`는 static 메서드만 있어 인스턴스 없이 바로 사용하고, 반올림 자릿수 조절은 곱하기/나누기로 구현한다.
11. ✅ `Scanner`는 공백/개행을 구분자로 순서대로 읽으며 타입을 맞춰야 한다.
12. ✅ `Random(seed)` 로 씨드를 고정하면 항상 같은 순서의 난수를 얻을 수 있다.
13. ✅ `java.time`의 월은 1~12로 직관적이며, `toEpochDay()`로 날짜 간격을 계산할 수 있다.
14. ✅ `ArrayList`는 크기가 자동으로 늘어나는 동적 배열이며, 제네릭으로 타입 안전성을 확보한다.

## 오늘 핵심 정리

**오전:**
```
String (불변)         → 상수 풀 공유, 루프 내 + 연산 주의
StringBuilder/Buffer  → 가변 버퍼, 빈번한 편집에 사용
```

**오후:**
```
Wrapper   → 기본형을 객체로 / parseInt()로 문자열 파싱
Math      → static 유틸리티 / 반올림 자릿수는 *10 → round → /10
Scanner   → 공백 기준 토큰 읽기 / 타입 순서 맞춰야 함
Random    → nextInt(N)으로 범위 지정 / 씨드 고정 시 항상 같은 난수
java.time → 불변 + 월이 1~12 / toEpochDay()로 날짜 차이 계산
ArrayList → 동적 배열 / 제네릭으로 타입 지정
```

**선택 기준 정리:**
- 문자열 단순 저장/비교 → `String`
- 문자열 빈번한 편집 → `StringBuilder`
- 크기 고정 데이터 → 배열
- 크기 가변 데이터 → `ArrayList`
