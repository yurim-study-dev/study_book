## 1. 상속의 필요성 - ch12.family

### 문제: 상속 없는 경우 (ex01)

세 개의 클래스(Parent, Son, Daughter)가 거의 똑같은 코드를 가짐:

```java
public class Parent {
    String name;
    Parent(String name) { this.name = name; }
    void eat() { System.out.println(name + " 음식을 맛있게 먹는다."); }
    void play() { System.out.println(name + " 등산한다."); }
    void sleep() { System.out.println(name + " 꿀잠을 잔다."); }
}

public class Son {
    String name;
    Son(String name) { this.name = name; }
    void eat() { System.out.println(name + " 음식을 맛있게 먹는다."); }
    void play() { System.out.println(name + " 게임을 한다."); }  // 다름
    void sleep() { System.out.println(name + " 꿀잠을 잔다."); }
}

public class Duaghter {
    String name;
    Duaghter(String name) { this.name = name; }
    void eat() { System.out.println(name + " 음식을 맛있게 먹는다."); }
    void play() { System.out.println(name + " 코노에 간다."); }  // 다름
    void sleep() { System.out.println(name + " 꿀잠을 잔다."); }
}
```

**문제점:**
- ❌ 중복 코드가 많음
- ❌ 같은 코드를 3번 반복
- ❌ 유지보수 어려움 (수정할 때 3곳 다 수정해야 함)

---

### 해결: 상속 사용 (ex02)

**부모 클래스:**
```java
public class Parent {
    String name;
    Parent(String name) { this.name = name; }
    void eat() { System.out.println(name + " 음식을 맛있게 먹는다."); }
    void play() { System.out.println(name + " 등산한다."); }
    void sleep() { System.out.println(name + " 꿀잠을 잔다."); }
}
```

**자식 클래스: Son**
```java
public class Son extends Parent {  // extends로 상속
    Son(String name) {
        super(name);  // 부모 생성자 호출
    }
    
    void play() {  // 메서드 오버라이딩
        System.out.println(name + " 게임을 한다.");
    }
    
    void work() {  // 자식만의 새로운 메서드
        System.out.println(name + " 일한다.");
    }
}
```

**자식 클래스: Daughter**
```java
public class Duaghter extends Parent {  // extends로 상속
    Duaghter(String name) {
        super(name);  // 부모 생성자 호출
    }
    
    void play() {  // 메서드 오버라이딩
        System.out.println(name + " 코노에 간다.");
    }
    
    void doFaceMask() {  // 자식만의 새로운 메서드
        System.out.println(name + " 마스크 팩을 한다.");
    }
}
```

**장점:**
- ✅ 공통 코드는 Parent에만 작성
- ✅ 자식은 다른 부분만 오버라이딩
- ✅ 유지보수 쉬움 (공통 부분 수정 시 한 곳만 수정)

---

### 상속 관계도

```
Parent (부모 클래스)
├─ name 필드
├─ eat() 메서드
├─ play() 메서드
└─ sleep() 메서드
    ↑ 상속
    ├─ Son (자식 클래스)
    │  └─ play() 오버라이딩
    │  └─ work() 추가
    │
    └─ Daughter (자식 클래스)
       └─ play() 오버라이딩
       └─ doFaceMask() 추가
```

---

### 메서드 오버라이딩

**정의:** 부모의 메서드를 자식이 다시 정의하는 것

```java
// 부모
void play() { System.out.println(name + " 등산한다."); }

// 자식 (오버라이딩)
void play() { System.out.println(name + " 게임을 한다."); }
```

**호출 결과:**
```java
Parent p = new Parent("부모");
p.play();  // "부모 등산한다."

Son s = new Son("아들");
s.play();  // "아들 게임을 한다."
```

## 2. 다형성(Polymorphism) - ch12.animal

### 부모 타입으로 자식 객체 참조

```java
public class Animal {
    void eat() { System.out.println("먹는다."); }
    void sleep() { System.out.println("잔다."); }
    void move() { System.out.println("스스로 움직인다."); }
}

public class Dog extends Animal {
    void eat() { System.out.println("강아지가 먹는다."); }  // 오버라이딩
    void bark() { System.out.println("멍멍 짖는다."); }
}

public class Cat extends Animal {
    void liquefy() { System.out.println("유리병에 들어갑니다."); }
}
```

### 다형성의 마법

```java
// ✅ 부모 타입으로 자식 객체를 참조 가능
Dog haru = new Dog();
Animal haru = new Dog();  // 이것도 가능!

// ✅ 메서드 호출 시 실제 객체의 메서드가 호출됨 (오버라이딩)
haru.eat();  // "강아지가 먹는다." (Dog의 eat 호출)
```

### 다형성을 활용한 통일된 처리

```java
static void animalCare(Animal a) {  // 부모 타입의 매개변수
    a.eat();
    a.move();
    a.sleep();
}

// 모든 동물을 같은 메서드로 처리!
animalCare(new Dog());   // ✅ Dog도 Animal이므로 가능
animalCare(new Cat());   // ✅ Cat도 Animal이므로 가능
```

### 주의: 잘못된 상속

```java
public class Notebook extends Animal {  // ❌ 노트북은 동물이 아님!
    void turnOn() { System.out.println("전원을 켠다."); }
}

// ❌ 논리적으로 맞지 않음
animalCare(new Notebook());
```

**원칙:**
```
is-a 관계가 성립해야 상속 가능!

✅ Dog is-a Animal (맞음)
✅ Cat is-a Animal (맞음)
❌ Notebook is-a Animal (틀림)
```

## 3. 핵심 개념

### extends 키워드

```java
public class 자식 extends 부모 {
    // 부모의 필드와 메서드를 상속받음
}
```

### super() - 부모 생성자 호출

```java
public class Son extends Parent {
    Son(String name) {
        super(name);  // Parent의 생성자를 호출
    }
}
```

### 메서드 오버라이딩 규칙

```
메서드 이름: 같아야 함
매개변수: 같아야 함 (개수, 타입, 순서)
반환 타입: 같아야 함
```

### 부모 타입 참조 변수

```java
Animal a1 = new Dog();      // ✅ 가능 (Dog는 Animal)
Animal a2 = new Cat();      // ✅ 가능 (Cat은 Animal)
Animal a3 = new Notebook(); // ❌ 불가능 (Notebook은 Animal이 아님)
```

---

## 배운점

1. ✅ 상속은 중복 코드를 제거하고 유지보수를 쉽게 한다.
2. ✅ `extends` 키워드로 상속 관계를 선언한다.
3. ✅ 자식 클래스는 부모의 필드와 메서드를 모두 상속받는다.
4. ✅ `super()`로 부모의 생성자를 호출할 수 있다.
5. ✅ 메서드 오버라이딩으로 자식이 부모의 메서드를 재정의할 수 있다.
6. ✅ 다형성(Polymorphism)은 부모 타입으로 다양한 자식 객체를 참조하는 것이다.
7. ✅ 다형성으로 통일된 인터페이스로 다양한 객체를 처리할 수 있다.
8. ✅ 상속 시 `is-a` 관계가 성립해야 한다.
9. ✅ 부모 타입 매개변수는 모든 자식 클래스의 객체를 받을 수 있다.
10. ✅ 메서드 호출 시 실제 객체의 메서드가 실행된다 (오버라이딩).

## 3. 심화: super와 super() - ch12.car 실습

### Car 부모 클래스

```java
public class Car {
    String model;
    int speed;
    int gear;
    
    Car(String model) {
        System.out.println("Car(String name) 생성자 호출됨.");
        this.model = model;
    }
    
    void speedUp() {
        speed += 10;
        System.out.println("가속합니다. 현재 속도는 " + speed + "Km/h 입니다.");
    }
    
    void speedDown() {
        speed -= 10;
        System.out.println("감속합니다. 현재 속도는 " + speed + "Km/h 입니다.");
    }
    
    void start() {
        System.out.println(model + ": 출발합니다.");
        gearUp();
        speedUp();
    }
    
    void stop() {
        System.out.println("멈춥니다.");
        gear = 0;
        while(speed > 0) {
            speedDown();
        }
    }
}
```

### Bus 자식 클래스 - super() 활용

```java
public class Bus extends Car {
    private int passengerCount;    // 승객 수
    private String no;              // 버스 번호
    private String type;            // 버스 종류
    private String[] stations;      // 노선
    private int station;            // 현재 위치
    private int price;              // 요금
    private int maxPassenger;       // 최대 승객수
    
    // super()로 부모 생성자 호출
    public Bus(String model, String no, String type,
               String[] stations, int price, int maxPassenger) {
        super(model);  // ⭐ 부모의 Car(String model) 생성자 호출
        this.no = no;
        this.type = type;
        this.stations = stations;
        this.price = price;
        this.maxPassenger = maxPassenger;
    }
    
    // 버스만의 기능들
    void ride() {
        if(passengerCount < maxPassenger) {
            passengerCount++;
            System.out.println("1명이 승차합니다.");
        } else {
            System.out.println("승차 인원이 초과되었습니다.");
        }
    }
    
    void leave() {
        passengerCount--;
        System.out.println("1명이 하차합니다.");
    }
    
    // ⭐ 메서드 오버라이딩 + super 활용
    void stop() {
        super.stop();  // 부모의 stop() 메서드 호출 (속도 0까지 감속)
        
        // 버스만의 추가 동작
        station++;
        if(station == stations.length) {
            station = 0;
        }
        System.out.println("이번역은 " + stations[station] + "역 입니다.");
    }
    
    String getBusInfo() {
        return "버스 번호: " + no
               + ", 종류: " + type
               + ", 현재 위치: " + stations[station]
               + ", 남은 좌석: " + (maxPassenger - passengerCount)
               + ", 요금: " + price;
    }
}
```

### 실제 사용 (BusTest)

```java
Bus mycar = new Bus("람보르기니", "10-2", "마을 버스",
                    new String[]{"우리집", "김포고등학교", "김포 시청"},
                    1300, 15);

mycar.start();     // Car의 start() 호출 (출발)
mycar.speedUp();   // Car의 speedUp() 호출 (가속) × 5회
mycar.stop();      // Bus의 stop() 호출 (부모 + 버스 로직)

mycar.ride();      // 승차 × 3회
mycar.start();

System.out.println(mycar.getBusInfo());  // 버스 정보 출력
```

### super() vs super 정리

| 구분 | super() | super |
|------|---------|-------|
| 용도 | 부모 생성자 호출 | 부모 필드/메서드 접근 |
| 위치 | 생성자 첫 줄 | 어디든 가능 |
| 예 | `super(model)` | `super.stop()` |

### super() 호출 순서

```
Bus 객체 생성
  ↓
Bus 생성자 호출
  ↓
super(model) 실행 ← 부모 Car 생성자 먼저 실행
  ↓
Car 생성자 완료 (model 초기화)
  ↓
Bus 생성자의 나머지 코드 실행
```

## 4. Static 멤버 (11.10)

### Static 필드 - 모든 인스턴스가 공유

```java
public class MathUtils {
    // static 필드: 모든 인스턴스가 공유
    public static final double PI = 3.14159;
    
    // static 메서드: 객체 생성 없이 호출
    public static int add(int a, int b) {
        return a + b;
    }
}

// 사용
MathUtils.PI;           // 인스턴스 생성 없이 바로 접근
int result = MathUtils.add(10, 20);  // 객체 생성 불필요
```

### Static 메서드의 제한사항

```java
public class Calculator {
    int instanceVal = 10;      // 인스턴스 변수
    static int staticVal = 20; // static 변수
    
    public static void staticMethod() {
        System.out.println(staticVal);   // ✅ static 변수 접근 가능
        // System.out.println(instanceVal);  // ❌ 에러! 인스턴스 변수 접근 불가
    }
    
    public void instanceMethod() {
        System.out.println(staticVal);    // ✅ 가능
        System.out.println(instanceVal);  // ✅ 가능
    }
}
```

**왜?** static 메서드는 클래스 로드 시점에 메모리에 올라가므로, 객체 생성 시점에 생성되는 인스턴스 필드/메서드에 접근 불가능!

## 5. 변수의 종류 (11.11)

### 세 가지 변수 비교

| 종류 | 선언 위치 | 저장 영역 | 생성 시점 | 소멸 시점 | 초기화 |
|------|---------|---------|---------|---------|--------|
| **지역 변수** | 메서드 내부 | 스택 | 선언 시 | 메서드 종료 시 | 필수 |
| **인스턴스 변수** | 클래스 내부 | 힙 | 객체 생성 시 | GC 수거 시 | 자동 |
| **클래스 변수** | 클래스 내부 (static) | 메서드 영역 | 클래스 로드 시 | 프로그램 종료 | 자동 |

### 메모리 다이어그램

```
메서드 영역
┌─────────────────────┐
│ static int classVar │ ← 모든 인스턴스 공유
└─────────────────────┘

힙
┌─────────────────────┐
│ int instanceVar 1   │ ← 객체1마다 독립
└─────────────────────┘
┌─────────────────────┐
│ int instanceVar 2   │ ← 객체2마다 독립
└─────────────────────┘

스택 (메서드 호출 시)
┌─────────────────────┐
│ int localVar        │ ← 메서드 종료 시 소멸
└─────────────────────┘
```

---

## 배운점 (추가)

11. ✅ static 멤버는 객체 생성 없이 클래스명으로 접근한다.
12. ✅ static 메서드는 인스턴스 필드/메서드에 직접 접근할 수 없다.
13. ✅ 세 가지 변수는 저장 영역, 생성/소멸 시점이 다르다.
14. ✅ `super()`는 부모 생성자를 호출하며, 생성자 첫 줄에만 위치한다.
15. ✅ `super.메서드()`로 부모의 메서드를 명시적으로 호출할 수 있다.
16. ✅ 오버라이딩된 자식 메서드에서 부모의 기능을 포함하고 싶으면 super를 사용한다.
