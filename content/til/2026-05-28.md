## 1. 배열 예외 처리 (Array Exception) - ch09

### 1-1. NullPointerException (NPE)

배열 참조 변수가 **null 상태**일 때 배열에 접근하려고 하면 발생합니다.

```java
int[] ageArr = null;  // 배열이 null

// System.out.println(ageArr[0]);  // ❌ NullPointerException 발생!
```

**예방 방법: null 체크**

```java
int[] ageArr = null;

if (ageArr != null) {           // null이 아닌지 먼저 확인
    System.out.println(ageArr[0]);
}
```

---

### 1-2. ArrayIndexOutOfBoundsException

배열의 범위를 벗어난 인덱스로 접근하려고 하면 발생합니다.

```java
String[] strArr = {"haru", "namu", "brong"};  // 길이 3

System.out.println(strArr[0]);   // ✓ "haru" 출력
System.out.println(strArr[3]);   // ❌ ArrayIndexOutOfBoundsException 발생!
```

## 2. 메서드 (Method) - ch10

### 10.1 메서드(Method)란?

**특정 기능을 수행하기 위해 관련된 코드들을 하나의 블록으로 묶어놓은 것**

필요할 때마다 호출해서 여러 번 재사용 가능합니다.

```java
// 메서드 없이
int sum1 = 10 + 100;
System.out.println("합계: " + sum1);

int sum2 = 20 + 100;
System.out.println("합계: " + sum2);

int sum3 = 30 + 100;
System.out.println("합계: " + sum3);

// 메서드로 간편하게
void printSum(int n1, int n2) {
    System.out.println("합계: " + (n1 + n2));
}

printSum(10, 100);
printSum(20, 100);
printSum(30, 100);
```

---

### 10.2 메서드를 사용하는 이유와 장점

#### 1️⃣ 코드의 재사용성(Reusability) 증대
자주 반복해서 사용하는 코드를 메서드로 한 번 정의해 두면, 필요할 때마다 호출하여 간편하게 다시 사용할 수 있어 개발 효율이 증대됩니다.

#### 2️⃣ 코드의 중복 제거
복사하여 붙여넣음으로써 발생하는 중복 코드를 제거하여 전체적인 프로그램 코드의 크기를 줄이고 깔끔하게 구성할 수 있습니다.

#### 3️⃣ 유지보수성(Maintainability) 향상
나중에 구현한 계산식이나 기능을 변경해야 할 때, 메서드 내부의 코드만 한 번 수정하면 해당 메서드를 사용하는 모든 곳에 수정 내용이 즉시 반영되므로 안전하고 빠릅니다.

#### 4️⃣ 가독성(Readability) 및 구조화
메서드 내부의 복잡한 구현 세부 코드는 신경 쓰지 않고 호출하는 방법만 알고 써도 됩니다. 프로그램을 큰 틀에서 흐름만 눈에 들어오기 때문에 전체 코딩 흐름을 한눈에 파악하기가 수월합니다.

---

### 10.3 메서드 선언과 호출

#### 10.3.1 메서드 선언

**형식:**

```java
[접근제어자] 반환타입 메서드명(매개변수타입 매개변수명, ...) {
    // 실행할 코드 블록
    return 반환값;  // 반환타입이 void가 아닐 때만 사용
}
```

**각 요소 설명:**

| 요소 | 설명 | 예시 |
|------|------|------|
| **접근제어자** | 메서드를 호출할 수 있는 범위 | public, private, protected, default |
| **반환타입** | 메서드가 반환할 데이터의 타입 | int, double, String, void |
| **메서드명** | 메서드의 이름 | add, getSum, printScore |
| **매개변수** | 메서드 호출 시 전달받는 값의 변수 | (int n1, int n2) |

---

#### 접근제어자 (Access Modifier)

```java
public int getSum(int n1, int n2) {
    // 어디서든 호출 가능
    return n1 + n2;
}

private void calculate() {
    // 같은 클래스 내부에서만 호출 가능
}

protected void method() {
    // 동일한 패키지 내의 클래스, 상속 관계의 자식 클래스에서 호출 가능
}

void defaultMethod() {
    // 동일한 패키지 내에 있는 클래스들에서만 호출 가능
    // 접근제어자 생략 시 기본값(default)
}
```

---

#### 반환타입 (Return Type)

```java
// 반환값 없음 (void)
void add() {
    int n1 = 10;
    int n2 = 100;
    System.out.println(n1 + " + " + n2 + " = " + (n1 + n2));
    // return; (생략 가능)
}

// 반환값 있음 (int)
int getSum(int n1, int n2) {
    int sum = n1 + n2;
    return sum;  // 반환타입과 반환값의 타입이 일치해야 함
}

// 반환값 있음 (double)
double getAverage(int[] scores) {
    int total = 0;
    for (int score : scores) {
        total += score;
    }
    return (double)total / scores.length;
}
```

---

#### 매개변수 (Parameter)

```java
// 매개변수 없음
void printHello() {
    System.out.println("Hello");
}

// 매개변수 1개
void add100(int n1) {
    System.out.println(n1 + " + 100 = " + (n1 + 100));
}

// 매개변수 여러 개
void printSum(int n1, int n2) {
    System.out.println(n1 + " + " + n2 + " = " + (n1 + n2));
}

// 매개변수 타입 다양
void printInfo(String name, int age, double score) {
    System.out.println("이름: " + name + ", 나이: " + age + ", 점수: " + score);
}
```

#### 10.3.2 메서드 호출

선언된 메서드를 실행하기 위해 메서드명과 필요한 **인자(Argument, 인수)**를 전달하여 호출합니다.

**형식:**

```java
메서드명(인자1, 인자2, ...);  // 반환값을 사용하지 않을 때

변수타입 변수명 = 메서드명(인자1, 인자2, ...);  // 반환값을 변수에 저장할 때
```

**예시:**

```java
add();                              // 인자 없이 호출
add100(20);                         // 인자 1개 전달
int result = getSum(10, 100);      // 반환값을 result에 저장
System.out.println(result);         // 110
```

**매개변수 vs 인자:**
- **매개변수 (Parameter)**: 메서드 정의 시 선언 → `void add100(int n1)`
- **인자 (Argument)**: 메서드 호출 시 전달 → `add100(20)`

### 10.4 main 메서드

자바 프로그램의 **시작점(Entry Point)** 역할을 하는 특수한 메서드입니다.

자바 가상머신(JVM)은 프로그램을 실행할 때 이 main 메서드를 찾아 실행합니다.

#### 10.4.1 전통적인 main 메서드 (Java 20 이전)

**형식:**

```java
public static void main(String[] args) {
    System.out.println("Hello World");
}
```

**각 키워드의 의미:**

| 키워드 | 의미 |
|--------|------|
| **public** | JVM이 어디서나 main 메서드에 접근 가능 |
| **static** | 클래스의 인스턴스 객체를 생성하지 않고도 호출 가능 |
| **void** | 프로그램 종료 시 반환값이 없음 |
| **String[] args** | 명령행에서 전달되는 문자열 매개변수들 (배열 형태) |

---

#### 10.4.2 Java 21 이후의 새로운 main 메서드 (간략화 문법)

**암묵적으로 선언된 클래스 (Implicitly Declared Classes / Unnamed Classes)**

```java
// 클래스 선언 생략, 파일 내부에 바로 메서드만 작성
void main() {
    System.out.println("New Hello World");
}
```

**특징:**
- 클래스 선언(`class ClassName { }`)을 생략하고 메서드만 작성
- 컴파일러가 소스코드 파일명을 클래스명으로 인식
- 외부에서 임포트(import)하여 사용 불가 (단독 실행용만 가능)

**인스턴스 메인 메서드 (Instance Main Methods)**

```java
// static과 public 생략, 명령행 매개변수 생략 가능
void main() {
    System.out.println("Instance main method");
}
```

**특징:**
- JVM이 내부적으로 클래스의 객체 인스턴스를 자동 생성 후 호출
- 클래스 내부의 다른 인스턴스 메서드를 `new` 연산자 없이 바로 호출 가능
- 명령행 매개변수가 필요 없으면 `String[] args` 생략 가능

---

### 10.5 명령행 매개변수 (Command Line Arguments)

프로그램을 실행할 때 **외부에서 값을 전달받아 사용**할 수 있는 매개변수입니다.

main 메서드의 매개변수 `String[] args` 배열을 통해 전달됩니다.

#### 사용 방법

**형식:**

```java
public static void main(String[] args) {
    if (args.length < 2) {
        System.out.println("사용법: java ch10.ArgsTest [이름] [나이]");
        return;  // 프로그램 종료
    }
    
    String name = args[0];                      // 첫 번째 인자
    int age = Integer.parseInt(args[1]);        // 두 번째 인자를 정수로 변환
    
    System.out.println("이름: " + name + ", 나이: " + age);
}
```

#### 실행 방식

**터미널에서:**

```bash
java ch10.ArgsTest haru 6
```

**args 배열에 저장되는 값:**
```
args[0] = "haru"
args[1] = "6"
```

#### 주요 메서드

| 메서드 | 설명 | 예시 |
|--------|------|------|
| `args.length` | 명령행 인자 개수 | if (args.length < 2) |
| `Integer.parseInt()` | 문자열을 정수로 변환 | int age = Integer.parseInt(args[1]); |
| `Double.parseDouble()` | 문자열을 실수로 변환 | double score = Double.parseDouble(args[0]); |

**주의:** 전달받은 데이터는 모두 **문자열(String)** 타입이므로, 숫자로 사용하려면 형변환이 필수입니다!

### 10.6 메서드 오버로딩 (Method Overloading)

동일한 이름을 가진 메서드를 한 클래스 내에 **여러 개 정의**하는 것입니다.

#### 오버로딩이 가능한 이유

원래 클래스 내에서 메서드 이름은 중복될 수 없지만, **매개변수의 개수나 타입이 서로 다르면** 동일한 이름의 메서드를 중복해서 정의할 수 있습니다.

```java
// ✓ 가능: 매개변수 타입 다름
void print(int num) { }
void print(double num) { }
void print(String text) { }

// ✓ 가능: 매개변수 개수 다름
void add(int n1) { }
void add(int n1, int n2) { }
void add(int n1, int n2, int n3) { }

// ✓ 가능: 매개변수 타입과 개수 다름
void calc(int a, int b) { }
void calc(double a, double b, double c) { }

// ❌ 불가능: 반환 타입만 다름 (컴파일 에러)
int getSum(int n1, int n2) { }
double getSum(int n1, int n2) { }  // 에러!
```

#### 오버로딩의 장점

메서드를 사용할 때 **데이터 타입에 관계없이 일관된 하나의 이름**만 사용할 수 있어 개발자의 편의성을 높입니다.

---

#### 실제 예시: toString 메서드

배열의 타입에 따라 다른 메서드를 호출하되, **같은 이름 toString** 사용:

```java
// int 배열을 문자열로 변환
String toString(int[] arr) {
    String result = "[ ";
    for (int i = 0; i < arr.length; i++) {
        result += arr[i];
        if (i != arr.length - 1) result += ", ";
    }
    result += " ]";
    return result;
}

// double 배열을 문자열로 변환
String toString(double[] arr) {
    String result = "[ ";
    for (int i = 0; i < arr.length; i++) {
        result += arr[i];
        if (i != arr.length - 1) result += ", ";
    }
    result += " ]";
    return result;
}

// String 배열을 문자열로 변환
String toString(String[] arr) {
    String result = "[ ";
    for (int i = 0; i < arr.length; i++) {
        result += arr[i];
        if (i != arr.length - 1) result += ", ";
    }
    result += " ]";
    return result;
}
```

**호출:**

```java
int[] scores = {89, 78, 79};
double[] avgScores = {12.0, 45.6};
String[] names = {"haru", "namu", "brong"};

System.out.println(toString(scores));     // [ 89, 78, 79 ]
System.out.println(toString(avgScores));  // [ 12.0, 45.6 ]
System.out.println(toString(names));      // [ haru, namu, brong ]
```

**컴파일러의 자동 선택:**
- `toString(scores)` → `int[]` 타입의 toString 호출
- `toString(avgScores)` → `double[]` 타입의 toString 호출
- `toString(names)` → `String[]` 타입의 toString 호출

실습: `ch10/MethodOverrding.java`

## 3. 메서드 문제 풀이 (오후 학습)

### 문제 상황

세 명의 점수를 받아서 각각의 **총점과 평균**을 구하는 프로그램:

```
haru: 80, 70, 56
namu: 90, 77, 80
brong: 100, 99, 88

각각의 총점과 평균을 구해서 출력하세요.
```

---

### 해결 방법: 메서드 설계

#### 메서드 1️⃣: sum - 총점 계산

```java
// 배열을 받아서 총점을 계산해서 반환
int sum(int[] scores) {
    int total = 0;
    
    for (int score : scores) {
        total += score;
    }
    
    return total;
}
```

---

#### 메서드 2️⃣: avg - 평균 계산

```java
// 배열을 받아서 평균을 계산해서 반환
double avg(int[] scores) {
    int total = sum(scores);           // sum 메서드 활용
    double scoreAvg = (double)total / scores.length;
    return Math.round(scoreAvg * 10) / 10.0;  // 소수점 한 자리
}
```

**Math.round() 사용법:**
```
Math.round(68.666 * 10) = 687
687 / 10.0 = 68.7 (소수점 한 자리)
```

---

#### 메서드 3️⃣: printScore - 결과 출력

```java
// 배열을 받아서 총점과 평균을 출력
void printScore(int[] scores) {
    int total = sum(scores);
    double scoreAvg = avg(scores);
    System.out.println("총점 : " + total + " , 평균 : " + scoreAvg);
}
```

---

### 전체 실행 코드

```java
int[] haruScore = {80, 70, 56};
int[] namuScore = {90, 77, 80};
int[] brongScore = {100, 99, 88};

printScore(haruScore);   // 총점 : 206 , 평균 : 68.7
printScore(namuScore);   // 총점 : 247 , 평균 : 82.3
printScore(brongScore);  // 총점 : 287 , 평균 : 95.7
```

## 배운점

1. ✅ 메서드는 특정 기능을 수행하는 재사용 가능한 코드 블록이다.
2. ✅ 메서드의 4가지 장점: 재사용성, 중복 제거, 유지보수성, 가독성 향상
3. ✅ 접근제어자로 메서드의 호출 범위를 제어한다 (public, private, protected, default).
4. ✅ 반환타입은 메서드가 돌려줄 데이터의 타입이다 (void일 수 있음).
5. ✅ 매개변수는 메서드 정의 시 선언, 인자는 호출 시 전달한다.
6. ✅ main 메서드는 프로그램의 시작점이며 JVM에 의해 자동으로 호출된다.
7. ✅ Java 21 이후 간략화된 main 메서드 문법을 사용할 수 있다.
8. ✅ 명령행 매개변수는 String[] args로 전달되며, 형변환이 필요하다.
9. ✅ 메서드 오버로딩은 같은 이름, 다른 매개변수로 여러 메서드를 정의하는 것이다.
10. ✅ 오버로딩은 매개변수의 타입이나 개수가 달라야 한다 (반환 타입 불가).
11. ✅ 컴파일러는 메서드 호출 시 매개변수를 보고 알맞은 메서드를 자동으로 선택한다.
12. ✅ 메서드 내에서 다른 메서드를 호출하여 코드를 재사용할 수 있다.

## 코드 정리

### 메서드 선언의 기본 구조

```java
[접근제어자] 반환타입 메서드명(매개변수타입 매개변수명, ...) {
    // 메서드 본문
    return 반환값;  // 반환타입이 void가 아닐 때만 필수

```

### 점수 계산 메서드

```java
// 총점
int sum(int[] scores) {
    int total = 0;
    for (int score : scores) {
        total += score;
    }
    return total;
}

// 평균 (소수점 한 자리)
double avg(int[] scores) {
    int total = sum(scores);
    double scoreAvg = (double)total / scores.length;
    return Math.round(scoreAvg * 10) / 10.0;
}

// 출력
void printScore(int[] scores) {
    int total = sum(scores);
    double scoreAvg = avg(scores);
    System.out.println("총점 : " + total + " , 평균 : " + scoreAvg);
}
```
