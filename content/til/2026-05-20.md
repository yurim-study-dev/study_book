## 1. 조건문이란?

조건식이 참(true)인지 거짓(false)인지에 따라 프로그램의 실행 흐름을 제어합니다.

프로그램이 의사결정을 할 수 있게 해줍니다.

---

## 2. if문

조건식이 참(true)일 때만 지정한 블록을 실행하고, 거짓(false)이면 실행하지 않고 건너뜁니다.

### 형식

```java
if (조건식) {
    // 조건식이 참일 때 실행할 구문
}
```

### 예시

```java
int score = 95;
if (score >= 90) {
    System.out.println("우수한 성적입니다."); // score가 95이므로 실행됨
}

score -= 10;
if (score >= 90) {
    System.out.println("우수한 성적입니다."); // score가 85이므로 실행되지 않음
}
```

---

## 3. if-else문

조건식이 참일 경우 if 블록을, 거짓일 경우 else 블록을 실행합니다.

### 형식

```java
if (조건식) {
    // 조건식이 참일 때 실행할 구문
} else {
    // 조건식이 거짓일 때 실행할 구문
}
```

### 예시

```java
int score = 75;
if (score >= 60) {
    System.out.println("합격입니다."); // 참일 때 실행
} else {
    System.out.println("불합격입니다."); // 거짓일 때 실행
}
```

---

## 4. if-else if문

조건식이 참일 경우 해당 구문을 수행하고, 거짓일 경우 else if 문의 조건식을 순차적으로 비교하여 참에 해당하는 구문을 수행합니다.

### 형식

```java
if (조건식1) {
    // 조건식1이 참일 때 실행할 구문
} else if (조건식2) {
    // 조건식2가 참일 때 실행할 구문
} else {
    // 모든 조건식이 거짓일 때 실행할 구문
}
```

### 예시

```java
int age = 19;
if (age >= 20) {
    System.out.println("성인");
} else if (age >= 14) {
    System.out.println("청소년"); // age가 19이므로 이 조건식이 충족되어 실행됨
} else {
    System.out.println("어린이");
}

age += 1;
if (age >= 20) {
    System.out.println("성인"); // age가 20이므로 이 조건식이 충족되어 실행됨
} else if (age >= 14) {
    System.out.println("청소년");
} else {
    System.out.println("어린이");
}
```

## 5. 중괄호 생략 규칙

if, else if, else 문 블록 내에 실행할 구문이 단 한 줄(하나의 실행문)일 경우, 중괄호 {}를 생략할 수 있습니다.

단, 생략할 경우 코드 가독성이 떨어질 수 있고, 나중에 실행 구문을 추가할 때 버그가 생기기 쉬우므로 실무에서는 가급적 중괄호를 명시적으로 작성하는 것을 권장합니다.

### 예시

```java
int score = 85;

// if, else if, else 블록 전체에서 실행문이 한 줄이므로 각각 중괄호 생략 가능
if (score >= 90)
    System.out.println("우수");
else if (score >= 80)
    System.out.println("보통"); // score가 85이므로 실행됨
else
    System.out.println("노력요함");
```

### 권장 방식 (중괄호 명시)

```java
int score = 85;

if (score >= 90) {
    System.out.println("우수");
} else if (score >= 80) {
    System.out.println("보통");
} else {
    System.out.println("노력요함");
}
```

## 6. switch문

switch 구문의 비교값과 같은 case 블록을 찾아서 실행합니다.

break 키워드를 만날 때까지 아래에 위치한 case 블록을 모두 실행합니다.

### 형식

```java
switch (비교대상) {
    case 값1:
        // 비교대상이 값1일 때 실행할 구문
        break;
    case 값2:
        // 비교대상이 값2일 때 실행할 구문
        break;
    default:
        // 매칭되는 값이 없을 때 실행할 구문
}
```

### 예시

```java
String role = "ADMIN";
switch (role) {
    case "ADMIN":
        System.out.println("관리자 권한"); // role이 "ADMIN"이므로 이곳으로 즉시 점프하여 실행
        break; // switch 문을 즉시 빠져나감
    case "USER":
        System.out.println("일반 사용자");
        break;
    default:
        System.out.println("권한 없음"); // 매칭되는 case가 없을 경우 실행
}

role = "USER";
switch (role) {
    case "ADMIN":
        System.out.println("관리자 권한");
        break;
    case "USER":
        System.out.println("일반 사용자"); // role이 "USER"이므로 이곳으로 즉시 점프하여 실행
        break;
    default:
        System.out.println("권한 없음");
}
```

### break 키워드 없을 때의 문제

break 키워드를 작성하지 않으면 아래에 위치한 다음 case 블록들까지 연달아 실행됩니다. (fall-through)

```java
String role = "USER";
switch (role) {
    case "ADMIN":
        System.out.println("관리자 권한");
        break;
    case "USER":
        System.out.println("일반 사용자"); // 실행됨
        // break가 없음!
    default:
        System.out.println("권한 없음"); // 이것도 실행됨 (fall-through)
}
```

## 7. if vs switch 선택 기준

### if문 사용하는 경우

- 범위 비교가 필요할 때
- 논리 연산자를 사용할 때
- 복잡한 조건이 필요할 때

```java
int score = 85;
if (score >= 90) {
    System.out.println("A");
} else if (score >= 80) {
    System.out.println("B");
} else if (score >= 70) {
    System.out.println("C");
}
```

### switch문 사용하는 경우

- 특정 값과의 정확한 일치를 비교할 때
- 경우의 수가 많을 때
- 코드가 더 깔끔해질 때

```java
int month = 4;
switch (month) {
    case 1, 3, 5, 7, 8, 10, 12 -> System.out.println("31일");
    case 4, 6, 9, 11 -> System.out.println("30일");
    case 2 -> System.out.println("28일");
}
```

## 배운점

1. 조건문은 프로그램이 의사결정을 할 수 있게 해준다.
2. if문은 조건이 참일 때만 코드를 실행한다.
3. if-else문으로 두 가지 경로 중 하나를 선택할 수 있다.
4. if-else if-else는 여러 조건을 순차적으로 검사한다.
5. switch문은 특정 값과의 정확한 일치 비교에 효과적이다.
6. switch문에서 break를 빠뜨리면 fall-through가 발생한다.
7. 중괄호는 한 줄의 코드일 때 생략 가능하지만, 실무에서는 명시적으로 작성하는 것을 권장한다.
8. 문제의 특성에 따라 if문과 switch문을 선택해서 사용한다.
9. 같은 문제를 여러 방식으로 풀 수 있고, 효율성이 다르다.

## 어제 학습과의 연결

조건문은 연산자에서 배운 비교 연산자(==, !=, >, <, >=, <=)와 논리 연산자(&&, ||, !)를 사용합니다.

예:
- `if (score >= 90)` - 비교 연산자
- `if (month < 1 || month > 12)` - 논리 연산자
- `if (age >= 14 && age < 20)` - 논리 연산자와 비교 연산자

조건식을 올바르게 작성하려면 연산자를 잘 이해해야 합니다.
