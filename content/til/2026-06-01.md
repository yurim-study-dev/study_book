## 3. 메서드 분석

### 메서드 1️⃣: fight() - 적과 싸우기

```java
static int[] fight(int hp, int enemyHp) {
    System.out.println("적과 싸웁니다.");
    
    // 적이 준 데미지 (0 ~ 10)
    int enemyDamage = (int)(Math.random() * 11);
    enemyHp -= enemyDamage;
    
    // 플레이어가 입은 데미지 (0 ~ 10)
    int damage = (int)(Math.random() * 11);
    hp -= damage;
    
    // 결과 출력
    System.out.println("적에게 " + enemyDamage + "의 데미지를 주고 " 
                     + damage + "의 데미지를 입었다.");
    System.out.println("나의 남은 체력은 " + hp + "입니다.");
    System.out.println("적의 남은 체력은 " + enemyHp + "입니다.");
    
    // 수정된 hp와 enemyHp를 배열로 반환
    return new int[]{hp, enemyHp};
}
```

**특징:**
- **배열 반환**: 두 개의 값(hp, enemyHp)을 배열로 반환
- **양방향 데미지**: 플레이어와 적 모두 데미지 입음
- **랜덤값**: Math.random() * 11 (0 ~ 10)

**호출 방식:**
```java
int[] result = fight(hp, enemyHp);
hp = result[0];           // 플레이어 HP 업데이트
enemyHp = result[1];      // 적 HP 업데이트
```

### 메서드 2️⃣: runAway() - 도망가기

```java
static int[] runAway(int hp) {
    // 도망 중 입을 데미지 (0 ~ 2)

    hp -= damage;
    
    // 새로운 적 생성 (HP 10 ~ 99)
    int enemyHp = (int)(Math.random() * 90) + 10;
    
    System.out.println("도망간다. 대신 " + damage + "의 체력이 소비되었다.");
    System.out.println("남은 체력은 " + hp + "입니다.");
    
    // 수정된 hp와 새로운 enemyHp를 반환
    return new int[]{hp, enemyHp};
}
```

**특징:**
- **배열 반환**: hp와 새로운 enemyHp 반환
- **새 적 생성**: 도망 후 새로운 적이 나타남
- **소량 데미지**: 도망 중 0 ~ 2의 데미지

### Switch 표현식 활용

```java
switch(action) {
    case 1 -> {
        // 싸우기
        int[] result = fight(hp, enemyHp);
        hp = result[0];
        enemyHp = result[1];
    }
    case 2 -> {
        // 도망가기
        int[] result = runAway(hp);
        hp = result[0];
        enemyHp = result[1];
    }
    case 3 -> {
        // HP 회복
        hp += 10;
        System.out.println("체력이 10 회복 되었다.");
    }
    case 4 -> {
        // 필살기
        if(ultimateSkill > 0) {
            int chance = (int)(Math.random() * 2);  // 0 또는 1
            enemyDamage = enemyHp * chance;          // 100% 또는 0%
            enemyHp -= enemyDamage;
            ultimateSkill--;
            System.out.println("궁극의 필살기를 사용했다.");
        } else {
            System.out.println("사용할 수 있는 필살기가 없습니다.");
        }
    }
}
```

**특징:**
- **Switch 표현식**: Java 12+ 문법 (->)
- **메서드 호출**: 게임 로직을 메서드로 분리
- **배열 반환값 활용**: fight()와 runAway() 결과 처리

```java
// 기존 switch 문

    case 1:
```

**Switch 표현식의 장점:**
- 코드가 더 간결함
- break 불필요

## CH11: 객체지향 프로그래밍(OOP) 기초

### 핵심 개념

**객체지향 프로그래밍(OOP)**은 데이터와 그 데이터를 처리하는 함수를 객체라는 하나의 단위로 묶어 관리하는 프로그래밍 패러다임입니다.

### 객체, 클래스, 인스턴스

```
객체(Object): 실제로 인지할 수 있는 실체 (자동차, 사람, 은행 계좌 등)
  └─ 속성(상태): 필드로 표현
  └─ 행위(동작): 메서드로 표현

클래스(Class): 객체를 만들기 위한 설계도 (메모리에 실체 없음)
  └─ 틀, 규격, 명세서

인스턴스(Instance): 클래스를 바탕으로 new 연산자로 생성된 실제 객체
  └─ 힙 메모리에 실제로 존재
  └─ 인스턴스 변수를 가지고 있음
```

### 클래스의 구성 요소

```java
public class Car {
  // 1. 필드 (객체의 속성/상태)
  String model;      // 모델명
  int speed;         // 속도
  
  // 2. 생성자 (객체 초기화)
  public Car(String model) {
    this.model = model;
    this.speed = 0;
  }
  
  // 3. 메서드 (객체의 행위/동작)
  public void accelerate() {
    this.speed += 10;
    System.out.println(model + " 속도 증가! 현재 속도: " + speed);
  }
}
```

### new 연산자와 객체 생성

```java
// 1. new 연산자로 힙 메모리에 객체 생성
Car myCar = new Car("현대 소나타");

// 2. 도트(.) 연산자로 필드와 메서드 접근
System.out.println(myCar.model);  // "현대 소나타" (필드 접근)
myCar.accelerate();                // 메서드 호출
```

**메모리 구조:**
```
스택:
┌─────────────┐
│ myCar → 참조값(주소)
└─────────────┘
         ↓
힙:
┌─────────────────┐
│ model: "현대 소나타"
│ speed: 0
└─────────────────┘
```

### 생성자 (Constructor)

객체가 **new로 생성될 때 가장 먼저 호출**되어 초기 설정을 담당합니다.

**특징:**
- 반환 타입이 없음 (void도 붙이지 않음)
- 메서드명 = 클래스명 (대소문자 정확히 일치)
- 매개변수를 받아 필드를 초기화

```java
public class Member {
  String id;
  String password;
  
  // 생성자 1: 기본 생성자
  public Member() {
    this.id = "guest";
    this.password = "0000";
  }
  
  // 생성자 2: 매개변수 있는 생성자 (오버로딩)
  public Member(String id, String password) {
    this.id = id;
    this.password = password;
  }
}

// 사용
Member m1 = new Member();                    // 기본 생성자 호출
Member m2 = new Member("user1", "1234");     // 오버로딩된 생성자 호출
```

### this와 this()

```java
public class User {
  String name;
  int age;
  
  // this: 자기 자신을 가리키는 참조
  public User(String name, int age) {

  }

  }

```

### 캡슐화와 접근 제어자

**캡슐화**: 중요한 데이터를 외부에서 임의로 변경하지 못하도록 은닉하는 기법

**접근 제어자 범위:**
```
private (가장 제한적)
  ↓
default (같은 패키지)
  ↓
protected (같은 패키지 + 상속 클래스)
  ↓
public (가장 개방적)
```

**Getter/Setter 패턴:**
```java
public class Account {
  private int balance;  // private으로 직접 접근 차단
  
  // Getter: 값 읽기
  public int getBalance() {
    return this.balance;
  }
  
  // Setter: 값 설정 (유효성 검사 가능)
  public void setBalance(int amount) {
    if (amount >= 0) {
      this.balance = amount;
    } else {
      System.out.println("음수는 불가능합니다.");
    }
  }
}
```

### 변수의 3가지 종류

| 종류 | 선언 위치 | 저장 영역 | 초기화 | 생성 시점 |
|------|---------|---------|--------|---------|
| **지역 변수** | 메서드 내부 | 스택 | 필수 | 변수 선언 시 |
| **인스턴스 변수** | 클래스 내부 (static 없음) | 힙 | 자동 | 객체 생성 시 |
| **클래스 변수** | 클래스 내부 (static 있음) | 메서드 영역 | 자동 | 클래스 로드 시 |

```java
public class VariableExample {

  static int count = 0;
  
  // 인스턴스 변수: 객체마다 독립적
  String name;
  
  public void method() {
    // 지역 변수: 메서드 실행 중에만 존재
    int localVar = 10;
  }
}
```

### static 멤버

static이 붙으면 클래스 멤버로, 객체를 생성하지 않고도 접근 가능합니다.

```java
public class MathUtils {
  // 정적 상수
  public static final double PI = 3.14159;
  
  // 정적 메서드 (인스턴스 생성 필수 없음)
  public static int add(int a, int b) {
    return a + b;
  }
}

// 사용
System.out.println(MathUtils.PI);         // 클래스명으로 직접 접근
int result = MathUtils.add(10, 20);       // 인스턴스 생성 불필요
```

3. ✅ new 연산자로 힙 메모리에 객체를 생성한다.

5. ✅ this는 자기 자신 객체를 가리키는 참조다.
6. ✅ 캡슐화는 private으로 필드를 보호하고 Getter/Setter로 접근을 제어한다.
7. ✅ 지역변수는 스택, 인스턴스변수는 힙, 클래스변수는 메서드 영역에 저장된다.
8. ✅ static 멤버는 모든 인스턴스가 공유하며 클래스명으로 직접 접근한다.
