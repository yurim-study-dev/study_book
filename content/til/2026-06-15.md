### 8번 - LIMIT 페이징 (핵심!)

1페이지당 5개씩, 3페이지 조회 → 11~15번째 행

```sql
-- 방법 1: LIMIT 시작위치, 개수
SELECT id, title, view_count FROM post
ORDER BY view_count DESC
LIMIT 10, 5;

-- 방법 2: LIMIT 개수 OFFSET 시작위치 (동일한 결과)
SELECT id, title, view_count FROM post
ORDER BY view_count DESC
LIMIT 5 OFFSET 10;
```

**페이지 계산 공식**

```
시작위치 = (페이지번호 - 1) × 페이지당 개수
3페이지  = (3 - 1) × 5 = 10
```

| 페이지 | LIMIT | 결과 |
|--------|-------|------|
| 1페이지 | LIMIT 0, 5 | 1~5번째 |
| 2페이지 | LIMIT 5, 5 | 6~10번째 |
| 3페이지 | LIMIT 10, 5 | 11~15번째 |

## 오전/오후: SQL 내장 함수 - 2.4

### 2.4.1 문자열 함수

```sql
-- CONCAT: 문자열 합치기
SELECT name, email, CONCAT(name, ' (', email, ')') AS member_info FROM member;

-- LENGTH: 바이트 길이 (한글 1자 = 3바이트)
SELECT email, LENGTH('hello'), LENGTH('안녕하세요'), LENGTH(email) AS email_length FROM member;

-- REPLACE: 문자열 치환
SELECT content, REPLACE(content, '안녕하세요', 'HI') AS replaced_content
FROM post WHERE content LIKE '%안녕하세요%';

-- SUBSTRING + CONCAT: 미리보기 자르기
SELECT title, CONCAT(SUBSTRING(title, 1, 10), '...') AS preview FROM post;

-- UPPER / LOWER: 대소문자 변환
SELECT email, LOWER(email) AS email_lower, UPPER(email) AS email_upper FROM member;

-- TRIM: 앞뒤 공백 제거
SELECT email, TRIM(email) AS trimmed_email FROM member;
```

### 2.4.2 날짜 함수

```sql
-- 현재 날짜+시간 / 날짜만
SELECT NOW() AS current_datetime;
SELECT CURDATE() AS curr_date;

-- DATE_FORMAT: 날짜 포맷 변환
SELECT id, title, DATE_FORMAT(created_at, '%Y년 %m월 %d일 %a %H시 %i분 %s초') AS write_date
FROM post;
```

**DATE_FORMAT 주요 포맷 코드**

| 코드 | 의미 | 예시 |
|------|------|------|
| `%Y` / `%y` | 4자리 / 2자리 연도 | 2026 / 26 |
| `%m` / `%c` | 2자리 / 1~2자리 월 | 06 / 6 |
| `%M` / `%b` | 영문 월 / 약식 | June / Jun |
| `%d` / `%e` | 2자리 / 1~2자리 일 | 15 / 15 |
| `%H` / `%h` | 24시간 / 12시간 시 | 14 / 02 |
| `%p` | AM/PM | PM |
| `%i` | 분 | 30 |
| `%s` | 초 | 00 |
| `%W` / `%a` | 영문 요일 / 약식 | Sunday / Sun |

```sql
-- DATE_SUB / DATE_ADD: 날짜 연산
-- 최근 7일 이내 가입 회원
SELECT id, name, created_at FROM member
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY);

-- 가입한지 1개월 지난 회원
SELECT * FROM member WHERE created_at < DATE_SUB(CURDATE(), INTERVAL 1 MONTH);

-- 복합 단위: 34일 12시간 경과 회원 조회
SELECT * FROM member
WHERE created_at < DATE_SUB(CURDATE(), INTERVAL '34 12' DAY_HOUR);

-- DATEDIFF: 날짜 차이(일수)
SELECT name, created_at, DATEDIFF(CURDATE(), created_at) AS days_since_join FROM member;
```

**DATE_SUB / DATE_ADD 단위 종류**

| 단위 | 예시 |
|------|------|
| YEAR, MONTH, DAY, HOUR, MINUTE, SECOND | `INTERVAL 7 DAY` |
| YEAR_MONTH | `INTERVAL '1-6' YEAR_MONTH` (1년 6개월) |
| DAY_HOUR | `INTERVAL '2 10' DAY_HOUR` (2일 10시간) |
| HOUR_MINUTE | `INTERVAL '12:30' HOUR_MINUTE` (12시간 30분) |

### 2.4.3 집계 함수

여러 행의 데이터를 하나의 결괏값으로 요약한다.

```sql
-- COUNT: 행 개수
SELECT COUNT(*) FROM member;           -- 전체 회원 수
SELECT COUNT(*) FROM post;             -- 전체 게시글 수

-- SUM / AVG / MIN / MAX
SELECT member_id,
    SUM(view_count)  AS total_views,
    AVG(view_count)  AS avg_views,
    MIN(view_count)  AS min_views,
    MAX(view_count)  AS max_views
FROM post
WHERE member_id = 3;
```

| 함수 | 설명 |
|------|------|
| `COUNT(*)` | 전체 행 개수 |
| `SUM(컬럼)` | 합계 |
| `AVG(컬럼)` | 평균 |
| `MAX(컬럼)` | 최댓값 |
| `MIN(컬럼)` | 최솟값 |

### 2.4.4 NULL 처리 함수

```sql
-- IFNULL: NULL이면 대체값 반환
SELECT name, IFNULL(phone, '미등록') AS phone FROM member;

-- COALESCE: NULL이 아닌 첫 번째 값 반환 (여러 컬럼 우선순위)
SELECT name, COALESCE(phone, email, '연락처 없음') AS contact FROM member;
```

---

### 2.4.5 흐름 제어 함수

```sql
-- IF: 단순 조건 분기
SELECT name, IF(phone IS NULL, '연락처 없음', '연락처 등록') AS phone_status FROM member;

-- CASE WHEN: 다중 조건 분기 (자바의 switch-case와 유사)
SELECT name, created_at,
    CASE
        WHEN created_at < DATE_SUB(NOW(), INTERVAL 1 YEAR)  THEN '우수 회원'
        WHEN created_at < DATE_SUB(NOW(), INTERVAL 1 MONTH) THEN '일반 회원'
        ELSE '신규 회원'
    END AS member_grade
FROM member;
```

## 오후: GROUP BY / HAVING - 2.5

### 2.5.1 GROUP BY - 그룹화

같은 값끼리 묶어서 집계 함수를 적용할 때 사용한다.

```
과일 판매 테이블 예시:
사과[3, 5] → SUM = 8
바나나[2, 1] → SUM = 3
```

```sql
-- 회원별 게시글 수
SELECT member_id, COUNT(*) AS post_count FROM post GROUP BY member_id;

-- 게시글별 댓글 수 (내림차순)
SELECT post_id, COUNT(*) AS reply_count FROM reply
GROUP BY post_id ORDER BY reply_count DESC;

-- 회원별 게시글 수 + 총 조회수
SELECT member_id, COUNT(*) AS post_count, SUM(view_count) AS total_views
FROM post GROUP BY member_id ORDER BY post_count, total_views DESC;

-- 가입 연도별 회원 수 (DATE_FORMAT 활용)
SELECT DATE_FORMAT(created_at, '%Y') AS join_year, COUNT(*) AS join_count
FROM member GROUP BY DATE_FORMAT(created_at, '%Y');
```

---

### 2.5.2 HAVING - 그룹 필터링

GROUP BY 이후 집계 결과에 조건을 적용한다. WHERE과 역할이 다르다.

```sql
-- 게시글 5개 이상인 회원만 조회
SELECT member_id, COUNT(*) AS post_count, SUM(view_count) AS total_views
FROM post
GROUP BY member_id
HAVING COUNT(*) >= 5
ORDER BY post_count, total_views DESC;
```

**WHERE vs HAVING 차이**

```
SQL 실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

WHERE  : 그룹화 전, 개별 행 필터링 → 집계 함수 사용 불가
HAVING : 그룹화 후, 집계 결과 필터링 → 집계 함수 사용 가능
```

```sql
-- WHERE: 개별 행 필터링 (그룹화 전)
SELECT member_id, COUNT(*) FROM post
WHERE member_id != 1       -- 1번 회원 제외 후 그룹화
GROUP BY member_id;

-- WHERE + HAVING 조합
SELECT member_id, COUNT(*) AS post_count FROM post
WHERE DATE_FORMAT(created_at, '%Y-%m') = '2026-06'   -- 6월 게시글만
GROUP BY member_id
HAVING COUNT(*) >= 2;                                 -- 2개 이상인 그룹만

-- 에러: WHERE에서 집계 함수 사용 불가
-- WHERE COUNT(*) >= 2  → 반드시 HAVING으로 이동!
```

---

### 2.5.3 GROUP BY 주의사항 - SELECT 절에 쓸 수 있는 컬럼

GROUP BY를 쓰면 SELECT에는 **GROUP BY 기준 컬럼** 또는 **집계 함수**만 쓸 수 있다.

```sql
-- 에러: title은 GROUP BY에 없어서 어떤 값을 표시할지 알 수 없음
SELECT member_id, title, COUNT(*) AS post_count
FROM post GROUP BY member_id;
```

**해결 방법 3가지**

```sql
-- 방법 1: 집계 함수 적용 (MAX, MIN 등)
SELECT member_id, MAX(title), COUNT(*) AS post_count FROM post GROUP BY member_id;

-- 방법 2: GROUP_CONCAT으로 여러 값을 하나의 텍스트로 합침
SELECT member_id, GROUP_CONCAT(title SEPARATOR ', ') AS titles, COUNT(*) AS post_count
FROM post GROUP BY member_id;

-- 방법 3: title도 GROUP BY 기준에 추가
SELECT member_id, title, COUNT(*) AS post_count FROM post GROUP BY member_id, title;
```

## 오후: DCL - 2.6

### 2.6.1 사용자 계정 관리

```sql
-- 계정 생성
CREATE USER 'user1'@'localhost' IDENTIFIED BY '1111';   -- 로컬 전용
CREATE USER 'user2'@'%'         IDENTIFIED BY '2222';   -- 모든 호스트 허용
CREATE USER 'board_app'@'board.likelion.net' IDENTIFIED BY '1234';  -- 특정 도메인

-- 비밀번호 변경
ALTER USER 'user1'@'localhost' IDENTIFIED BY '3333';

-- 계정 삭제
DROP USER 'user2'@'localhost';
```

| 호스트 지정 | 설명 |
|------------|------|
| `localhost` | 서버 로컬에서만 접속 가능 |
| `%` | 모든 호스트에서 접속 가능 |
| 특정 IP/도메인 | 지정된 위치에서만 접속 가능 |

### 2.6.2 권한 조회 (SHOW GRANTS)

```sql
SHOW GRANTS;                                -- 현재 접속 계정 권한 조회
SHOW GRANTS FOR 'user1'@'localhost';        -- 특정 계정 권한 조회
```

---

### 2.6.3 권한 부여 (GRANT)

```sql
-- 테이블 단위: user1에게 member 테이블 SELECT, INSERT 권한 부여
GRANT SELECT, INSERT ON member TO 'user1'@'localhost';

-- 데이터베이스 단위: board_db 전체 SELECT 권한
GRANT SELECT ON board_db.* TO 'user1'@'localhost';

-- 애플리케이션 계정에 CRUD 권한 부여
GRANT SELECT, INSERT, UPDATE, DELETE ON board_db.* TO 'board_app'@'board.likelion.net';
```

**권한 적용 범위**

| 범위 | 문법 |
|------|------|
| 모든 DB | `*.*` |
| 특정 DB 전체 | `board_db.*` |
| 특정 테이블 | `board_db.member` |

---

### 2.6.4 권한 회수 (REVOKE)

```sql
-- user1의 member 테이블 INSERT 권한 회수
REVOKE INSERT ON member FROM 'user1'@'localhost';
```

---

### 2.6.5 권한 그룹 (ROLE)

여러 사용자에게 동일한 권한을 효율적으로 부여·회수하기 위해 권한을 그룹으로 묶어 관리한다.

```sql
-- 1. 역할 생성
CREATE ROLE 'developer';

-- 2. 역할에 권한 부여
GRANT SELECT, INSERT, UPDATE, DELETE ON board_db.* TO 'developer';

-- 3. 사용자에게 역할 부여
GRANT 'developer' TO 'user1'@'localhost';

-- 4. 로그인 시 자동 활성화 설정
SET DEFAULT ROLE 'developer' TO 'user1'@'localhost';
```

## 배운점

1. ✅ 페이징 공식: `시작위치 = (페이지-1) × 개수`. `LIMIT 10, 5`와 `LIMIT 5 OFFSET 10`은 동일하다.
2. ✅ `LENGTH`는 바이트 기준이라 한글 1자는 3바이트로 계산된다.
3. ✅ `DATE_FORMAT`의 포맷 코드 `%Y/%m/%d %H:%i:%s`를 외워두면 다양한 형태로 날짜를 표현할 수 있다.
4. ✅ `IFNULL`은 단순 NULL 대체, `COALESCE`는 여러 컬럼 중 첫 번째 non-NULL 값을 반환한다.
5. ✅ `CASE WHEN`은 SQL의 다중 분기 조건문으로 자바의 switch-case와 유사하다.
6. ✅ `GROUP BY`는 같은 값끼리 그룹을 묶어 집계 함수를 적용할 때 사용한다.
7. ✅ `WHERE`는 그룹화 전 개별 행 필터링, `HAVING`은 그룹화 후 집계 결과 필터링이다.
8. ✅ `GROUP BY`에 없는 일반 컬럼을 SELECT에 쓰면 에러 → 집계 함수나 GROUP_CONCAT으로 해결.
9. ✅ DCL에서 `GRANT`는 권한 부여, `REVOKE`는 회수. `ROLE`로 여러 권한을 그룹 관리할 수 있다.
10. ✅ 호스트를 `localhost`로 지정하면 로컬 전용, `%`로 지정하면 모든 외부 접속을 허용한다.

## 오늘 핵심 정리

```
페이징 공식
  시작위치 = (페이지 - 1) × 페이지당 개수
  LIMIT 시작위치, 개수  =  LIMIT 개수 OFFSET 시작위치

집계 함수
  COUNT(*) / SUM / AVG / MAX / MIN

NULL 처리
  IFNULL(값, 대체값)              → NULL이면 대체값
  COALESCE(값1, 값2, '기본값')   → NULL 아닌 첫 번째 값

조건 분기
  IF(조건, 참, 거짓)
  CASE WHEN 조건 THEN 결과 ... ELSE 기본 END

GROUP BY / HAVING
  WHERE  → 그룹화 전 행 필터링 (집계 함수 사용 불가)
  HAVING → 그룹화 후 집계 결과 필터링 (집계 함수 사용 가능)
  실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

GROUP BY SELECT 규칙
  SELECT에는 GROUP BY 컬럼 또는 집계 함수만 가능
  그 외 컬럼 → MAX/MIN 또는 GROUP_CONCAT 사용

DCL
  CREATE USER '이름'@'호스트' IDENTIFIED BY '비밀번호'
  GRANT  권한 ON 객체 TO 계정   → 권한 부여
  REVOKE 권한 ON 객체 FROM 계정 → 권한 회수
  CREATE ROLE / GRANT 역할 TO 계정 → 권한 그룹 관리
```
