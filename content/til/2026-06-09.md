## 오전: 추상 클래스 (Abstract Class) - 13.3

### 추상 클래스란?

**정의:**
- 실체 클래스들의 공통적인 특성을 추출하여 선언한 미완성 설계도
- 객체를 직접 만들 수 없음 (단독 인스턴스 생성 불가)
- `abstract` 키워드로 선언

```java
abstract class Parent {
    // 추상 클래스
}

// ❌ new Parent(); 불가능!
```

### 추상 클래스의 특징

1. **직접 객체 생성 불가**
   ```java
   // ❌ 에러 발생
   Parent p = new Parent();
   
   // ✅ 자식 클래스를 통해서만 생성
   Parent p = new Son("이름");
   ```

2. **생성자는 존재** (부모 생성자 호출용)
   ```java
   abstract class Parent {
       Parent(String name) {  // 생성자 존재
           this.name = name;
       }
   }
   ```

3. **일반 메서드도 가능**
   ```java
   abstract class Parent {
       void eat() {  // 일반 메서드 (구현부 있음)
           System.out.println("먹는다");
       }
   }
   ```

---

### 추상 메서드 (Abstract Method)

**정의:**
- 선언부만 있고 구현부(바디)가 없는 메서드
- `abstract` 키워드 사용
- 자식이 반드시 오버라이딩해야 함

```java
abstract class Parent {
    // ⭐ 추상 메서드: 구현부가 없음
    abstract void study();
}
```

### 추상 메서드를 강제하는 이유

```
문제:
┌─────────────────────────────┐
│ 일반 부모 클래스            │
│ - play() 메서드 정의       │
└─────────────────────────────┘
        ↓
자식이 play()를 오버라이드하지 않아도 됨 (문법상 문제 없음)
        ↓
❌ 버그 가능성! (자식만의 특화된 동작을 구현 못함)

해결책:
┌─────────────────────────────┐
│ 추상 클래스 Parent          │
│ - abstract void study()    │
└─────────────────────────────┘
        ↓
자식이 반드시 study()를 구현해야 함 (컴파일러가 강제)
        ↓
✅ 안전함! (필수 메서드 구현 보장)
```

---

### 실습 예제: ch13.family.ex02

#### Parent (추상 클래스)

```java
public abstract class Parent {
    String name;
    
    Parent(String name) {
        this.name = name;
    }
    
    // 일반 메서드
    void eat() {
        System.out.println(name + ": 음식을 먹는다.");
    }
    
    // ⭐ 추상 메서드: 자식이 반드시 구현해야 함
    abstract void study();
}
```

#### Son (자식이 추상 메서드 구현)

```java
public class Son extends Parent {
    Son(String name) {
        super(name);
    }
    
    // ⭐ 추상 메서드 구현 (필수!)
    @Override
    public void study() {
        System.out.println(name + ": 도서관에서 공부한다.");
    }
    
    // 오버라이딩
    @Override
    void play() {
        System.out.println(name + ": 게임을 한다.");
    }
}
```

#### Daughter (자식이 추상 메서드 구현)

```java
public class Daughter extends Parent {
    Daughter(String name) {
        super(name);
    }
    
    // ⭐ 추상 메서드 구현 (필수!)
    @Override
    public void study() {
        System.out.println(name + ": 과외를 받는다.");
    }
}
```

## 오후: 인터페이스 (Interface) - 13.4

### 인터페이스란?

**정의:**
- 객체의 사용 설명서이자 극단적인 추상화 설계도
- 구현부 없이 **규격(스펙)만** 정의
- `interface` 키워드로 선언

```java
interface Chargeable {
    void charge();  // 구현부 없음 (자동으로 public abstract)
}
```

---

### 인터페이스의 문법적 특징

1. **interface 키워드 사용**
   ```java
   public interface Chargeable {
       // ...
   }
   ```

2. **모든 필드는 상수 (public static final)**
   ```java
   interface Calculator {
       double PI = 3.14159;  // 자동으로 public static final
   }
   ```

3. **메서드는 추상 메서드** (구현부 없음)
   ```java
   interface Chargeable {
       void charge();  // 자동으로 public abstract void charge();
   }
   ```

4. **다중 구현 지원** (핵심!)
   ```java
   // 여러 인터페이스를 한 번에 구현
   class ElectricCar extends SuperCar implements Chargeable, Drivable {
       // ...
   }
   ```

5. **객체 생성 불가**
   ```java
   // ❌ 에러
   Chargeable c = new Chargeable();
   
   // ✅ 인터페이스를 구현한 클래스로 생성
   Chargeable c = new ElectricCar(...);
   ```

---

### 추상 클래스 vs 인터페이스

| 구분 | 추상 클래스 | 인터페이스 |
|------|-----------|---------|
| **관계** | Is-A (혈연관계) | Can-Do (능력) |
| **목적** | 상태 + 행위 공유 | 행동 규격만 정의 |
| **다중** | 불가능 (extends 1개) | 가능 (implements 다중) |
| **필드** | 일반 필드 가능 | 상수만 (public static final) |
| **메서드** | 일반 메서드 가능 | 추상 메서드 (Java 8+는 default도 가능) |

---

### 실습 예제: ch13.supercar

#### SuperCar (부모 클래스)

```java
public class SuperCar {
    int zero100;  // 0~100km 소요 시간
    
    SuperCar(int zero100) {
        this.zero100 = zero100;
    }
    
    void move() {
        System.out.println("슈퍼카가 100미터를 달리는데 " 
                          + zero100 + " 초 걸렸다.");
    }
}
```

#### Chargeable (인터페이스)

```java
public interface Chargeable {
    void charge();  // 충전 기능 규격만 정의
}
```

#### ElectricCar (상속 + 다중 구현)

```java
// ⭐ SuperCar를 상속 + Chargeable을 구현
public class ElectricCar extends SuperCar implements Chargeable {
    ElectricCar(int zero100) {
        super(zero100);
    }
    
    // SuperCar의 move() 오버라이딩
    @Override
    void move() {
        super.move();  // 부모 동작 먼저 실행
        System.out.println("전기를 사용합니다.");
    }
    
    // Chargeable의 charge() 구현
    @Override
    public void charge() {
        System.out.println("전기를 충전합니다.");
    }
}
```

#### GasolineCar (일반 상속만)

```java
public class GasolineCar extends SuperCar {
    GasolineCar(int zero100) {
        super(zero100);
    }
    
    @Override
    void move() {
        super.move();
        System.out.println("기름을 사용합니다.");
    }
}
```

#### HybridCar (일반 상속만)

```java
public class HybridCar extends SuperCar {
    HybridCar(int zero100) {
        super(zero100);
    }
    
    @Override
    void move() {
        super.move();
        System.out.println("기름과 전기를 사용합니다.");
    }
}
```

#### Driver (다형성 활용)

```java
public class Driver {
    // SuperCar 타입으로 모든 자식 클래스를 받을 수 있음 (다형성)
    void drive(SuperCar car) {
        car.move();
    }
}
```

#### DriveTest (테스트)

```java
public class DriveTest {
    public static void main(String[] args) {
        GasolineCar gcar = new GasolineCar(8);
        HybridCar hcar = new HybridCar(7);
        ElectricCar ecar = new ElectricCar(4);
        PlugHybridCar pcar = new PlugHybridCar(8);
        
        Driver driver = new Driver();
        
        // 같은 메서드로 다양한 자동차를 운전 (다형성)
        driver.drive(gcar);   // 기름 사용
        driver.drive(hcar);   // 기름 + 전기 사용
        driver.drive(ecar);   // 전기 사용
        driver.drive(pcar);   // 플러그 하이브리드
        
        // ElectricCar는 Chargeable이므로 충전 가능
        ecar.charge();  // 전기를 충전합니다.
    }
}
```

## 오후 2부: 자바 API 문서 - 14.1

### 자바 API 문서란?

**정의:**
- 자바 표준 라이브러리의 모든 클래스, 메서드의 상세 명세서
- 공식 온라인 문서: https://docs.oracle.com/en/java/javase/

### 공식 문서 구조

```
1. Hierarchy (상속 계층도)
   - 클래스의 부모가 누구인지

2. Field Summary (필드 요약)
   - private, public, static 여부
   - final 여부

3. Constructor Summary (생성자 요약)
   - 어떤 매개변수를 받는지

4. Method Summary (메서드 요약)
   - Modifier: public, static, final 등
   - Return Type: 반환 타입
   - Method Description: 메서드 설명 및 매개변수
```

### IDE에서 API 활용

**IntelliJ IDEA:**
- 마우스를 메서드명에 올리면 설명 팝업 표시
- Windows: `Ctrl + Q` 또는 `F1` (Mac)

```java
String str = "Hello";
str.length();  // ← 여기서 Ctrl+Q를 누르면 API 문서 표시
```

## CH14: 자바 주요 패키지와 API - 14.2

### java.lang 패키지

```
특징: import 없이 자동으로 사용 가능!

주요 클래스:
- Object: 모든 클래스의 최상위 부모
- String: 문자열 처리
- System: 시스템 관련 기능
- Math: 수학 연산
```

### java.util 패키지

```
특징: 명시적 import 필수!

주요 클래스:
- Scanner: 입력 받기
- Random: 난수 생성
- ArrayList: 동적 배열
- HashMap: 키-값 저장소
- Date, Calendar: 날짜/시간
```

### import 예시

```java
// java.lang은 import 불필요
String str = "Hello";
System.out.println(str);

// java.util은 import 필요
import java.util.Scanner;
import java.util.ArrayList;

Scanner sc = new Scanner(System.in);
ArrayList<String> list = new ArrayList<>();
```

## 배운점

1. ✅ 추상 클래스는 미완성 설계도이며 객체를 직접 생성할 수 없다.
2. ✅ 추상 메서드는 자식이 반드시 구현하도록 강제한다.
3. ✅ 인터페이스는 행동 규격만 정의하는 계약서이다.
4. ✅ 다중 구현을 통해 하나의 클래스가 여러 능력을 가질 수 있다.
5. ✅ 추상 클래스 (Is-A)와 인터페이스 (Can-Do)는 용도가 다르다.
6. ✅ 다형성으로 부모 타입 참조변수가 자식 객체를 참조할 수 있다.
7. ✅ 공식 API 문서는 자바 클래스 사용의 가이드북이다.
8. ✅ java.lang은 import 없이, java.util은 import 필요하다.
